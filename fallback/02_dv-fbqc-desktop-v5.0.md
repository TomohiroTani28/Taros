## セクション1: v4.0 R6からv5.0への主要変更概要

### 1.1 設計哲学の転換

v4.0 R6は「段階的小型化」（Phase 0→1→2で60m²→2.5m²→2m²）を方針としていた。v5.0は**「デスクサイドファースト設計」**に転換する。

**v4.0の設計フロー**: 大型研究機 → 小型化 → デスクサイド
**v5.0の設計フロー**: デスクサイド筐体制約から逆算 → 必要技術を特定 → 実現可能性を検証

この転換が可能になった理由:
1. PT205（8.6kg冷却器）の商用化により、冷却系のサイズ制約が解消
2. WI-SNSPD 99.73%により、検出器数を1/6に削減可能
3. InP QD 1550nmの急速な進展により、QFC系の完全排除が射程内
4. Floquet符号のフォトニック専用最適化により、物理qubit数を半減
5. PsiQuantum OmegaによりGlobalFoundries製造プロセスが確立

### 1.2 v5.0の六原則

v4.0の五原則に「Desktop Deployability」を追加し六原則とする。

| # | 原則 | v4.0表現 | v5.0表現 |
|---|------|---------|---------|
| 1 | フォールトトレランス | 既報告技術の組合せ | 同左 |
| 2 | **Desktop Deployability** | Phase別床面積目標 | **60cm×40cm×50cm筐体に全機能を収容** |
| 3 | モジュラー拡張性 | Phase間モジュール共通化>60% | **ワンチップモジュールの枚数追加で線形拡張** |
| 4 | 光ネイティブ設計 | 冷却系の段階的簡素化 | **PT205単体で全冷却要件を充足** |
| 5 | 実用性重視 | Phase別BoM | **量産100台での単価$420K** |
| 6 | **QFC-Free設計** | QFC除去はPhase 2オプション | **InP QD 1550nm直接発光を主パスに昇格** |

## セクション3: 光源系設計 — QFC-Free InP QD 1550nm

### 3.1 InP QD 1550nm直接発光の採用

v4.0 R6ではInP QD 1550nmをPhase 2のGo/No-Goオプション（C43）としていたが、2026年2月のarXiv:2602.06140の結果を受けてv5.0では**Primary光源に昇格**する。

#### 3.1.1 最新実績（2026年2月時点）

| パラメータ | v4.0 R6 Go基準 | **2026年2月実績** | 判定 |
|-----------|---------------|-----------------|------|
| HOM visibility V | >0.95 | **0.90 ± 0.03** | 未達（あと5pt） |
| g²(0) | <0.02 | **0.005 ± 0.001** | **達成** |
| 輝度 | >0.40 | 0.30-0.35 (推定) | 未達（改善中） |
| 方式 | — | TPE (二光子共鳴励起) + Purcell増強 | — |
| 基板 | — | InAs/InAlGaAs QD + Si wafer | CMOS互換 |

#### 3.1.2 V>0.95達成への技術ロードマップ

2026年2月のV=0.90からV>0.95への改善は以下の3経路で実現可能:

| 改善策 | V改善量 | 実現時期 | 根拠 |
|--------|--------|---------|------|
| Purcellファクター向上（F=4→10） | +3pt | 2027 Q1 | 共振器Q値最適化 |
| スペクトルフィルタリング（エタロン） | +2pt | 即時適用 | GaAs QDで実証済み手法 |
| 電場安定化（DC-Stark雑音抑制） | +1pt | 2027 Q2 | 能動フィードバック |
| **合計** | **+6pt → V=0.96** | **2027 Q2** | |

**結論**: 2027年中盤にGo基準（V>0.95）達成の確率は**65%**と評価する。

## セクション4: 検出系設計 — 超高効率WI-SNSPD

### 4.1 WI-SNSPD 99.73%の採用

v4.0 R6ではWI-SNSPD（導波路統合型SNSPD）をPhase 2目標（効率>99%）としていたが、2025年に南京大学+北京大学チームが**オンチップ検出効率99.73%**を実証した。v5.0ではこれをDesktop Prototypeに採用する。

#### 4.1.1 コム型ナノワイヤの革新

従来のWI-SNSPDはナノワイヤを導波路と平行に配置していたが、新方式では**ナノワイヤを導波路に対して横向き（transverse）に配置したコム（comb）形状**を採用。

```
従来方式:         新方式（コム型）:

━━━━━━━━━━━━     ┃ ┃ ┃ ┃ ┃ ┃ ┃
                  ┃ ┃ ┃ ┃ ┃ ┃ ┃
（ナノワイヤ ‖ 導波路）  （ナノワイヤ ⊥ 導波路）
                  ━━━━━━━━━━━━━
```

さらに、**2つのコム型検出器を1導波路上にカスケード配置**し、1段目を透過した光子を2段目で検出。2段合計で99.73%。

#### 4.1.2 製造方式: ハイブリッド膜転写

SNSPDを**フレキシブル膜として製造し、PIC上に転写**する方式。Transfer Printing技術とシナジーがある。

| 製造ステップ | 内容 | 歩留まり |
|------------|------|---------|
| 1. NbN膜成膜 | Mist CVD (R6 C47) on sacrificial Si wafer | 膜厚均一性±1% |
| 2. コム型パターニング | KrFステッパ (R6 C47) | 解像度100nm |
| 3. 膜リフトオフ | 犠牲層エッチング→フレキシブルNbN膜 | 95% |
| 4. 膜転写 | PDMSスタンプでSiN PIC上に転写 | 93% |
| 5. アニール+検査 | 2.5Kで検出効率測定 | 90% |
| **全体歩留まり** | | **79%** |

500個の有効素子に必要な製造数: 500/0.79 ≈ **633個** → 冗長込み700個製造

#### 4.1.3 Desktop検出系のスペック

## セクション5: 符号設計 — Floquet-GB符号

### 5.1 Floquet-GB [[72,12,6]] の採用

v5.0のDesktop専用符号として**Floquet-GB [[72,12,6]]**を採用する。これはv4.0 R6のGB [[72,12,6]]（C49）をFloquet化したものである。

#### 5.1.1 符号パラメータ

| パラメータ | 値 | v4.0 R6 GB [[144,12,12]]比 |
|-----------|-----|--------------------------|
| [[n, k, d]] | [[72, 12, 6]] | 物理qubit半分、論理qubit同数 |
| 符号レート | 16.67% | **2倍** |
| Single-Shot | **可能**（Floquet化、confinement証明済み） | 同等 |
| サイクル時間 | **2μs（1回測定）** | 同等 |
| Tゲートレート | **3.5MHz** | 同等 |
| スタビライザ重み | ≤6 | 同等 |
| 1サイクルのfusion数 | ~260 | ~580の**半分以下** |
| SNSPD必要数 | ~500 | ~1,000の**半分** |

#### 5.1.2 Floquet化の原理

GB符号のcyclic対称性を利用し、スタビライザ測定をA/B/C 3つの群に分割して周期的に実行:

```
サイクル1: ── A測定 ── B測定 ── C測定 ──
サイクル2: ── A測定 ── B測定 ── C測定 ──
...

各群の測定は2体測定（fusion）の組み合わせで実行。
Single-Shot性: 1サイクル（A+B+C）で十分な情報を取得。
```

#### 5.1.3 FBQC融合ネットワークへの埋め込み

npj Quantum Information (2026) の "Fusion-based implementation of qLDPC codes with quantum emitters" に基づき、GB符号のFBQC埋め込みは以下の手順で行う:

1. **Tanner graph抽出**: GB [[72,12,6]]の安定化子行列からTanner graphを構築
2. **CSS構造の利用**: GB符号はCSS符号であり、X安定化子とZ安定化子を独立に融合ネットワークに変換
3. **量子エミッタベースRSG**: 各データqubitを決定論的エミッタ（InP QD）が生成するリソースステートに対応
4. **Type-II fusion配置**: Tanner graphの各辺をType-II fusionに変換、PIC上のMZIに物理マッピング
5. **時間多重**: cyclic対称性を利用し、インターリービング深度4で同一ハードウェアを時間方向に再利用

#### 5.1.4 erasure閾値の推定

Floquet-GB [[72,12,6]]のerasure閾値はStimシミュレーションで確定すべきだが、以下の推定が可能:

| 符号 | erasure閾値 | 根拠 |
|------|-----------|------|
| 表面符号 (MWPM) | 24.9% | Stace+ 2009 |
| Honeycomb Floquet (photonic) | **6.3% (損失閾値)** | arXiv:2410.07065 |
| GB [[144,12,12]] (推定) | ~20-25% | スタビライザ重み≤6 |
| **Floquet-GB [[72,12,6]] (推定)** | **~18-22%** | Floquet化+小距離d=6 |
| PsiQuantum adaptive encoded | **18.8%** | PsiQuantum 2025 |

v5.0 Desktopのε_herald (ヘラルド後erasure率):
```
【全経路η（参考）】: η_full = 0.93 (PIC全体) × 0.95 (遅延線) × 0.997 (WI-SNSPD) = 0.881
  → 全経路erasure = 1 - 0.881² × 0.97 = 24.7% (符号閾値との比較には不適切)

【正式: ヘラルド後η】: η_post_herald = 0.966 (PIC fusion後段) × 0.977 (遅延線) × 0.997 (WI-SNSPD) = 0.942
  → ε_herald = 1 - 0.942² × 0.97 = 1 - 0.861 = 13.9% (§9.3と整合)
```

**注**: erasure閾値との比較はヘラルド後損失のみ(§9.2-9.3)が正しい定義。ε_herald=**13.9%**。

**結論**: ε_herald=13.9%は基本閾値18-22%に対し**マージン4-8pt**。追加対策なしでも動作可能。

**追加マージン（オプション）**:
1. **Belief Matching + Soft Information**: 閾値を+4-6pt改善 → 実効閾値22-28%
2. **相関erasureデコーディング**: 閾値を+3-5pt改善
3. **合計実効閾値: ~28-34%** → ε_herald 13.9%に対し**マージン14-20pt** (§9.3と整合)

| WI-SNSPD | -0.01 | 0.997 |
| **合計（ヘラルド後）** | **-0.26** | **0.942** |

### 9.3 erasure率計算

```
ε_herald = 1 - η_post_herald² × P_fusion_success
         = 1 - 0.942² × 0.97
         = 1 - 0.861
         = 0.139 (13.9%)

ε_herald(v5.0) = 13.9%
```

| 構成 | ε_herald | 閾値（推定） | マージン |
|------|---------|------------|---------|
| v4.0 R6 Phase 1 | ~15-21% | ~28-34% (GB+BM+Soft+相関) | 7-19pt |
| **v5.0 Desktop** | **13.9%** | **~28-34%** | **14-20pt** |

**結論**: v5.0 Desktopのε_herald=13.9%は、あらゆる合理的な閾値推定に対して**十分なマージン**を持つ。

### 9.4 η×V²統合効率指標

```
η = 0.587 (end-to-end)
V = 0.95 (InP QD 2027目標)
η×V² = 0.587 × 0.9025 = 0.530

V = 0.97の場合（GaAs QD + QFC構成のフォールバック）:
η = 0.53 × 0.9409 = 0.499
```

InP QD (QFC-free) はV=0.95でもη改善効果が大きく、GaAs QD + QFC (V=0.97)よりη×V²で**6%優位**。

## 1. ビジョンと定義

### 1.1 「デスクサイド量子コンピュータ」とは

**定義**: 一般的なサーバールームやラボに設置可能な、42U標準ラック1本以内に全機能を収容した誤り訂正型量子コンピュータ。

```
┌─────────────────────────────────────────────┐
│                                             │
│   ┌─────────┐                               │
│   │         │  42U Standard Rack            │
│   │  Taros  │  600mm x 1070mm x 2000mm     │
│   │ Deskside│                               │
│   │   QC    │  + 外部冷却ユニット             │
│   │         │    400mm x 400mm x 800mm      │
│   │         │                               │
│   └─────────┘  ┌────────┐                   │
│                │ Cooling │                   │
│                │  Unit   │                   │
│                └────────┘                   │
│                                             │
│   設置面積: 2.0m x 1.5m = 3.0m²             │
│   消費電力: < 5kW                            │
│   重量: < 120kg（冷却ユニット含む）            │
│   冷却水: 不要（空冷コンプレッサ）              │
│                                             │
└─────────────────────────────────────────────┘
```

### 1.2 なぜ「デスクサイド」が正しい目標か

| 観点 | 根拠 |
|------|------|
| **市場** | 量子コンピュータのオンプレミス需要（金融・製薬・防衛）は2030年代前半に$10B規模と予測。クラウド専用では満たせないレイテンシ・セキュリティ要件が存在する |
| **技術的到達性** | Phase 2（2031-2032）で1ラック+外部冷却が既に設計済み。追加2-3年の技術成熟で「空冷・冷却水不要」が射程圏内 |
| **競争優位** | 超伝導方式は原理的にmK冷却が必要で、デスクサイドは不可能。フォトニック方式のみが実現可能な形態であり、決定的差別化要因 |
| **2045年目標との整合** | 20年ビジョンの「個人量子デバイス」への中間マイルストーンとして、2035年のデスクサイドQCは不可欠 |

### 1.3 超伝導方式との構造的差異（小型化の物理的根拠）

```
超伝導方式（IBM/Google）:
  量子ビット動作温度: 10-20 mK  → 希釈冷凍機が必須（~1,000kg, ~200kW）
  配線: 同軸ケーブル数千本      → 冷凍機サイズが配線で律速
  デスクサイド化: 物理的に不可能（冷凍機だけで1トン超）

フォトニック方式（Taros）:
  量子ビット（光子）: 室温動作   → 冷却不要
  光学回路: 室温動作             → 冷却不要
  冷却対象: SNSPDのみ（2.5-4.2K）→ PT冷凍機で十分（40-75kg）
  配線: 光ファイバ（軽量）       → 重量・体積の律速要因にならない
  デスクサイド化: 技術的に到達可能
```

## 2. デスクサイドQCのスペック目標

### 2.1 ハードウェアエンベロープ

| 項目 | 目標値 | 制約根拠 |
|------|--------|---------|
| **設置面積** | ≤ 3.0 m² | 標準ラック+冷却ユニット+保守スペース |
| **ラック** | 42U × 1本 | 標準サーバールーム互換 |
| **重量** | ≤ 120 kg（ラック+冷却） | エレベーター搬入可能、床荷重300kg/m²以内 |
| **消費電力** | ≤ 5 kW | 一般的な単相200V/30Aで運用可能 |
| **冷却水** | **不要**（空冷コンプレッサ） | 水配管工事の排除が設置容易性の鍵 |
| **振動** | VC-C以下（12.5 μm/s rms） | 一般オフィスビル相当。VC-D不要 |
| **騒音** | ≤ 55 dBA @ 1m | サーバールーム許容範囲 |
| **設置時間** | ≤ 3日（搬入+接続+較正） | 顧客エンジニアの最小拘束 |
| **保守** | リモート監視 + 年2回オンサイト | PT冷凍機メンテ20,000h間隔（~2.3年） |

### 2.2 計算性能目標

| 項目 | 目標値 | 根拠 |
|------|--------|------|
| **論理qubit数** | 48-100 | GB [[144,12,12]]（Floquet）で42-100、最小構成でも12 |
| **論理エラー率** | p_L < 10⁻⁶ / gate | 量子化学計算の実用要件 |
| **Tゲートレート** | 3.5 MHz | Floquet Single-Shot（2μsサイクル） |
| **クリフォードゲートレート** | ~71 kHz（d=7） | d × 2μs |
| **end-to-end η** | ≥ 0.70（保守）/ 0.77（楽観） | クライオPIC構成による損失最小化 |
| **アプリケーション** | VQE（LiH, STO-3G）: ~100ms / QAOA（MaxCut n=50）: ~10ms | 50-100論理qubit範囲の実用問題 |

### 2.3 コスト目標

| 項目 | 目標値 |
|------|--------|
| **製造原価** | ≤ $1.0M（≈1.5億円） |
| **販売価格** | $1.5-2.0M（≈2.3-3.0億円） |
| **年間保守費** | $100-150K（≈1,500-2,300万円） |
| **量産時目標（100台/年）** | 製造原価 $500K以下 |

## 4. 小型化の技術階段（Phase 2 → デスクサイドQC）

### 4.1 Phase 2（2031-2032）からの差分

Phase 2の1ラック+外部冷却構成を基盤とし、以下の追加技術でデスクサイドQCを完成させる。

| 技術課題 | Phase 2（2031-2032） | デスクサイドQC（2033-2035） | 差分 |
|----------|---------------------|--------------------------|------|
| **冷却水** | 水冷チラー必要 | **空冷コンプレッサ** | チラー排除 |
| **QFC** | LNOI QFC η=0.75-0.82 | **On-chip QFC or QFC除去** | InP QD 1550nm直接 or 統合QFC |
| **SNSPD接続** | ファイバ結合 | **WI-SNSPD完全統合** | 結合損失消滅 |
| **遅延線** | 外部HCF + オンチップ | **クライオHCFスプール検討** | フィードスルー削除 |
| **制御** | FPGA×1 + ASIC | **SoC統合（FPGA+ASIC+CPU）** | ボード数1/3 |
| **較正** | ACP 3層（半自動） | **ACP 3層（全自動）+ ML自己較正** | 人的介入最小化 |
| **筐体** | ラック+外部冷却 | **一体型筐体（オプション）** | 搬入・設置の簡素化 |

### 4.2 技術成熟マトリクス

各技術の現在のTRLとデスクサイドQCに必要なTRL。

| 技術 | 現在TRL (2026) | Phase 2必要TRL | デスクサイドQC必要TRL | 成熟確率 | Critical Path |
|------|---------------|---------------|---------------------|---------|---------------|
| GaAs QD (925nm) | 4-5 | 6 | 7 | 85% | |
| Transfer Printing | 3-4 | 5 | **7** | 60% | **Yes** |
| LNOI+SiN統合PIC | 3-4 | 5 | **7** | 70% | **Yes** |
| WI-SNSPD | 3 | 5 | **7** | 55% | **Yes** |
| PT冷凍機(空冷) | 7 | 7 | **8** | 90% | |
| MKID読み出し | 3-4 | 5 | 6 | 65% | |
| デコーダASIC | 2-3 | 5 | 6 | 70% | |
| DNANF HCF | 4 | 5 | 6 | 80% | |
| Floquet Single-Shot | 3 | 5 | 6 | 75% | |
| GB [[144,12,12]] | 2-3 | 5 | 6 | 70% | |
| 自動較正(ACP) | 2 | 4 | **6** | 75% | |
| InP QD 1550nm直接 | 2-3 | — | 5 (option) | 40% | |

**Critical Path技術（3つ）**: Transfer Printing、LNOI+SiN統合PIC、WI-SNSPD
→ この3技術のうち2つ以上が成功すればデスクサイドQCは実現可能。

### 4.3 統合確率評価

| シナリオ | 前提条件 | デスクサイドQC実現確率 | 論理qubit数 |
|----------|---------|---------------------|-----------|
| **Best** | 3 Critical Path技術すべて成功 | 40% | 100+ |
| **Likely** | 3技術中2つ成功 | 65% | 48-72 |
| **Minimum** | 3技術中1つのみ成功 | 80% | 12-24 |
| **Fallback** | 3技術すべて失敗 | — | デスクサイド不可、2ラック構成に後退 |

3技術すべて失敗する確率: (1-0.60)×(1-0.70)×(1-0.55) = 0.40×0.30×0.45 = **5.4%**
→ **デスクサイドQCが何らかの形で実現する確率: ~95%**

## 6. 小型化の5つの技術レバー（定量分析）

### 6.1 レバー1: 冷却系の空冷化

| 項目 | Phase 2 | デスクサイドQC | 改善 |
|------|---------|--------------|------|
| 冷却方式 | PT冷凍機 + 水冷チラー | PT冷凍機 + **空冷コンプレッサ** | チラー排除 |
| 冷却水 | 必要（15L/min） | **不要** | 設置工事大幅削減 |
| コンプレッサ消費電力 | 2.5kW | 3.0kW（空冷のため微増） | +0.5kW |
| コンプレッサ重量 | 30kg + チラー20kg = 50kg | 35kg（一体型） | -15kg |
| 設置面積（冷却系） | 0.16m² + チラー0.25m² | **0.20m²** | -0.21m² |

**実現要件**: SNSPD数を1,500個以下に抑え、4Kステージ熱負荷を0.8W以下にすること。MKID読み出し（熱負荷65mW）の導入が前提。

**リスク**: 空冷コンプレッサの騒音が55dBAを超過する可能性（対策: 防音エンクロージャ+$10K）。

### 6.2 レバー2: WI-SNSPD + クライオPIC統合

| 項目 | Phase 2 | デスクサイドQC | 改善 |
|------|---------|--------------|------|
| SNSPD結合方式 | ファイバ結合 | **導波路集積（WI-SNSPD）** | 結合損失 0.29dB→0.04dB |
| SNSPD-PIC間接続 | ファイバ×1,500本 | **オンチップ（接続0本）** | 配線99.9%削減 |
| 4K→室温間配線 | MKID RF <10本 | MKID RF <10本 | 同等 |
| η（検出段） | 0.92 | **0.99** | +7.6% |
| 全系η | 0.70 | **0.77** | 3技術成功時 |

### 6.3 レバー3: QFC除去（InP QD 1550nm直接発光）

| 項目 | Phase 2（QFCあり） | デスクサイドQC（QFC除去時） | 改善 |
|------|-------------------|--------------------------|------|
| QFC損失 | 0.75-0.82 | **1.0（QFCなし）** | +1.2-2.5dB回復 |
| 全系η | 0.70 | **0.80+** | 顕著な改善 |
| 部品点数 | QFCチップ×8 + ポンプレーザ×4 | **削除** | -12部品 |
| コスト削減 | — | **-$120K** | QFC+ポンプレーザ分 |
| ラック占有 | 2U | **0U** | 2U空き |

**ただしInP QD 1550nmの成熟確率は40%**（2026年時点でg²(0)<0.01かつV>0.95の同時達成未報告）。Go/No-Go判定はPhase 1末（2030年）。

QFC除去のGo判定: InP QD単体で V > 0.93 かつ g²(0) < 0.02 を達成した場合。
No-Go時: On-chip QFC（LNOI導波路をPIC上に統合）で対応。η=0.82は確保。

### 6.4 レバー4: DNANF + クライオHCFスプール

| 項目 | Phase 2 | デスクサイドQC | 改善 |
|------|---------|--------------|------|
| HCF種類 | NANF | **DNANF** | 損失1/3、曲げ半径1/3 |
| スプール直径 | 10cm | **7cm**（DNANF曲げ半径5cm） | -30% |
| スプール配置 | ラック内（室温） | **クライオ内（4.2K）オプション** | フィードスルー削除 |
| 全スプール体積 | 4U | **3U** | -1U |

### 6.5 レバー5: 制御系SoC統合

| 項目 | Phase 2 | デスクサイドQC | 改善 |
|------|---------|--------------|------|
| 制御ボード | FPGA×1 + ASIC×1 + CPU×1 | **SoC 1チップ** | ボード3枚→1枚 |
| 消費電力 | 0.5kW | **0.2kW** | -60% |
| ラック占有 | 4U | **2U** | -2U |

**SoCアーキテクチャ**: Xilinx Versal AI EdgeのようなFPGA+ARM+AIエンジン統合チップに、デコーダASICをチップレット実装。2033年時点のプロセスノード（3-5nm）で十分実現可能。

## 8. 設置環境要件

### 8.1 要件比較（Phase 0 → デスクサイドQC）

| 要件 | Phase 0 | デスクサイドQC | 改善 |
|------|---------|--------------|------|
| 床面積 | 60m²（専用室） | **3m²（サーバールーム内）** | **95%削減** |
| 天井高 | ≥3.0m | ≥2.5m（標準天井） | 一般オフィス対応 |
| 床荷重 | 500kg/m²（局所1000） | **300kg/m²** | 一般オフィスビル対応 |
| 室温 | 22±0.5℃ | 22±2.0℃ | 精密空調不要 |
| 湿度 | 40-60% RH | 30-70% RH | 一般空調で対応 |
| 振動 | VC-D（半導体fab相当） | **VC-C以下** | 一般オフィスビル対応 |
| 電源 | 3相200V, 60kVA | **単相200V, 30A (6kVA)** | 一般電源で対応 |
| 冷却水 | 60L/min | **不要** | 水配管工事不要 |
| クリーンレベル | ISO 7推奨 | **不要**（密閉型光路） | クリーンルーム不要 |
| EMI | RF機器持込制限 | 通常のサーバールーム | 特別な制限不要 |

**設置工事**: 電源コンセント1口 + LANケーブル1本で接続完了（理想形態）。

### 8.2 「箱出し」設置のための設計原則

1. **光路完全密閉**: すべての光路をPIC内またはコネクタ付きファイバで接続。端面露出なし → クリーンルーム不要
2. **自動アライメント**: Transfer Printing + WI-SNSPDにより個別光学調整ゼロ
3. **自己較正**: ACP 3層が起動時に全パラメータを自動較正。ML推論で経年劣化を補償
4. **リモート監視**: 全センサーデータをクラウドに送信。異常検知→自動退避→オンサイト保守依頼

## 9. リスクと対策

### 9.1 技術リスク

| リスク | 確率 | 影響 | 対策 | 検証時期 |
|--------|------|------|------|---------|
| WI-SNSPD歩留<80% | 30% | SNSPD数不足→論理qubit削減 | ファイバ結合SNSPDにフォールバック（+0.5ラック） | Phase 2 |
| 空冷PT振動>5μm | 25% | SNSPD結合不安定 | WI-SNSPDなら振動無関係（導波路統合のため）。ファイバ結合時はバッファガス対策 | Phase 2.5初期 |
| InP QD 1550nm No-Go | 60% | QFC除去不可 | On-chip QFCで対応（η=0.82確保）。コスト+$80K | Phase 1末 |
| デコーダASIC遅延 | 20% | FPGA継続使用（+0.3kW, +1U） | FPGA性能で十分。ASICは次世代での最適化 | Phase 2 |
| DNANF曲げ半径の長期信頼性 | 15% | NANF使用（スプール体積1.5倍） | Phase 2初期でDNANF加速寿命試験 | Phase 2 |

### 9.2 フォールバック構成

デスクサイドQCが目標スペックを達成できない場合のフォールバック。

| レベル | 構成 | 床面積 | 論理qubit | 条件 |
|--------|------|--------|----------|------|
| **A（目標）** | 42U×1 + 外部冷却 | 3.0m² | 48-100 | 3 Critical Path技術中2以上成功 |
| **B（縮退）** | 42U×1 + 外部冷却大型化 | 4.0m² | 24-48 | WI-SNSPD失敗→ファイバ結合SNSPD |
| **C（最小）** | 42U×1 + ハーフラック + 外部冷却 | 5.0m² | 12-24 | 統合PIC失敗→ディスクリートPIC |

レベルCでもPhase 0の60m²から**92%削減**であり、超伝導方式に対する圧倒的優位は維持。

## 13. 20年ビジョンとの接続

```
2026-2028   Phase 0     最小論理qubit実証（60m², ラボ）
    │
2029-2030   Phase 1     小型実用機（5-6m², サーバールーム）
    │
2031-2032   Phase 2     次世代小型機（4m², サーバールーム）
    │
2033-2035 ★ Phase 2.5   デスクサイドQC（≤3m², 一般オフィス）  ← 本書の主軸
    │
2036-2038   Phase 3     マルチノード（デスクサイドQC ×10ノード = 1,000論理qubit）
    │
2039-2042   Phase 4     量子光チップ（モノリシック集積、チップ単価~1万円）
    │
2043-2045   Phase 5     個人量子デバイス（スマートフォンサイズ、室温動作SNSPD）
```

**デスクサイドQCはPhase 3のビルディングブロック**: マルチノード構成では、デスクサイドQC同士を光ファイバで接続する。10台のデスクサイドQC（各48-100論理qubit）をエキスパンダーグラフ接続すれば、480-1,000論理qubitの分散量子コンピュータが実現する。これはフォトニック方式のネイティブな利点（光ファイバ接続の容易さ）を最大限に活用するアーキテクチャである。

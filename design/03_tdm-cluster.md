## 1. クラスタ状態トポロジーの選択

### 1.1 候補比較

| トポロジー | GKP+表面符号適合性 | 生成複雑度 | TDM効率 | 選定 |
|-----------|---------|---|---------|---|
| Bilayer square lattice | 中 | 低（実証済み） | 高 | 次点 |
| **Macronode lattice** | **高** | **中（実証済み）** | **高** | **採用** |
| Raussendorf lattice (3D) | 最高 | 高（未実証） | 中 | 将来 |
| Brickwork state | 低 | 低 | 高 | 不採用 |

### 1.2 Macronode lattice採用の根拠

1. **GKP+表面符号の閾値最適化**: Stafford-Menicucci-Walshe (2025) により、macronode上のGKP+表面符号が最も低い閾値（σ ≈ 7.5dB with postselection）を示す
2. **2D universality**: macronode latticeは2Dでも普遍的量子計算が可能（各macronodeが4モードで構成）
3. **TDM生成の効率**: OPA 2本 + BS 3個 + 遅延ループ 2本で生成可能
4. **実証実績**: Asavanant+ (2019, Science) で100万モード級のmacronode latticeを実証済み

## 2. Macronode TDM生成回路

### 2.1 光学回路図

```
                    ┌─────────────────────────────────────────────────┐
                    │              TDMクラスタ生成モジュール (2U)         │
                    │                                                   │
 OPA#1 ══PMF══►    │    ┌────┐     ┌────┐                             │
 (squeezing)       │ ──►│BS₁ │────►│BS₃ │──────────────────►  out_A   │ → Homodyne A
                   │    │50:50│    │50:50│         ↑                   │
 OPA#2 ══PMF══►    │ ──►│    │──┐ │    │──┐      │                   │
 (squeezing)       │    └────┘  │ └────┘  │      │                   │
                   │            │         │      │                   │
                   │            ▼         ▼      │                   │
                   │     ┌──────────┐  ┌──────────────────┐           │
                   │     │Delay τ₁  │  │   Delay τ₂       │           │
                   │     │(短遅延)   │  │   (長遅延)        │           │
                   │     │= 1/f_rep │  │   = N_col/f_rep  │           │
                   │     └──────┬───┘  └────────┬─────────┘           │
                   │            │               │                     │
                   │            ▼               │                     │
                   │     ┌────┐                 │                     │
                   │     │BS₂ │◄────────────────┘                     │
                   │     │50:50│                                       │
                   │     │    │──────────────────────────►  out_B     │ → Homodyne B
                   │     └────┘                                       │
                   │                                                   │
                   └─────────────────────────────────────────────────┘
```

### 2.2 動作原理

1. **OPA#1, #2**: PPLN導波路OPAからパルスドスクイーズド真空 (100MHz) (σ ≈ 13dB) を生成
2. **BS₁ (50:50)**: 2つのスクイーズドモードをエンタングル → EPR対の連続ストリーム
3. **Delay τ₁ (短遅延)**: 1クロック分遅延。隣接時間モード間のエンタングルメント生成
4. **Delay τ₂ (長遅延)**: N_col クロック分遅延。2Dクラスタの「列方向」結合
5. **BS₂, BS₃ (50:50)**: 遅延後の再結合 → 2D macronode lattice完成
6. **out_A, out_B**: 2つの出力モードでホモダイン測定 → 測定型量子計算実行

### 2.3 生成されるクラスタ状態

```
時間方向 →
     ┌──●──●──●──●──●──●──●──●──●──●──●──  (列1)
     │  │  │  │  │  │  │  │  │  │  │  │
列方向 ●──●──●──●──●──●──●──●──●──●──●──●──  (列2)
 ↓   │  │  │  │  │  │  │  │  │  │  │  │
     ●──●──●──●──●──●──●──●──●──●──●──●──  (列3)
     │  │  │  │  │  │  │  │  │  │  │  │
     └──●──●──●──●──●──●──●──●──●──●──●──  (列N_col)
         (周期境界条件)

各 ● = 1 macronode = 4 physical modes
Macronode内の4モード: input_A, input_B から分配
```

> **開境界条件の実装**: 表面符号は開境界(smooth/rough boundary)を必要とする。TDMの長遅延ループは位相的に周期的だが、境界モードのBS操作を変更（測定で切断）することにより開境界を実現する。具体的には、round境界の1モード(N_col中の+1 boundary mode)で行・列の端を測定消去し、有効的な開境界とする。
> **検証状況**: Stafford-Menicucci-Walshe 2025 (PRL 134, 100602) は周期境界TDMを前提としている。開境界への変更が閾値に与える影響は<0.1dBと見込むが、Phase -1 T0b Stimシミュレーションで開境界・周期境界の両方でp_L(d=7)を比較し確認する。macronode内4モード(input_A, input_B, output_A, output_B)のうちinput_A/Bは固定X基底測定(FF不要)、output_A/Bのみ適応測定(FF-1適用)。この構造がFF-1 3モード遅延と整合することはMenicucci 2014 (PRL 113, 130501)に基づく。

## 3. 設計パラメータ

### 3.1 クロック・遅延パラメータ

| パラメータ | 値 | 根拠 |
|-----------|-----|------|
| OPAクロック (f_rep) | **100 MHz** | パルスOPA (Option B) 方式。GS-DFB 100MHz繰り返し |
| クロック周期 (T_clk) | 10 ns | 1/f_rep |
| 短遅延 τ₁ | **10 ns = 2.0m (PMF)** | 1クロック分 |
| 列数 N_col | **98 (=2d²)** | 表面符号d=7: 14行×7列=98 macronodes/round (data 49 + ancilla 48 + 1 boundary = 98) |
| 長遅延 τ₂ | **980ns = 200.2m PMF** (n_g=1.468, 精密切断; 概数~1μs/~200m) | N_col × T_clk = 98 × 10ns |
| 1フレーム = N_col × N_row | 98 × N_row | N_rowは計算時間に応じて可変 |
| **論理qubit数** | **N_row / (2d²)** | d=7: 98モード/論理qubit |

### 3.2 論理qubit数 vs 計算時間

TDMの決定的優位: ハードウェア不変で論理qubit数を増減。

| 論理qubit数 | N_row | 計算時間 | ハードウェア変更 |
|------------|-------|---------|--------------|
| 1 | 98 | 9.8μs | なし |
| 10 | 980 | 98μs | なし |
| 50 | 4,900 | 490μs | なし |
| 100 | 9,800 | 980μs | なし |
| **1,000** | **98,000** | **9.8ms** | **なし** |

### 3.3 遅延線の物理設計

| 遅延線 | 方式 | 長さ | サイズ | 損失 | 設置場所 |
|--------|------|------|--------|------|---------|
| 短遅延 τ₁ | PMFスプール | 2m | φ5cm×2cm | 0.004dB | モジュール内 |
| **長遅延 τ₂** | **PMF (偏波保持ファイバ)** | **200m** | **φ15cm (ラック) / φ8cm (Portable)** | **0.07dB** | 筐体内 |
| (代替) 長遅延 τ₂ | DNANF | 200m | φ15cm×5cm | 0.013dB | 偏波問題あり |

**PMFを長遅延に採用。

**ファイバ長精密要件**: PMF群速度n_g=1.468において、遅延10ns=2.04m、遅延980ns=200.2m (d=7, N_col=98)。
文中の"2m""200m"は概数。実装時はクロック同期のため精密切断+PZT微調整でN_col×T_clk遅延を実現する。切断精度要件: ±50μm (精密クリーバー使用、対応する時間誤差±0.25ps) [PZTストローク±50μmと整合するよう±1mm→±50μmに厳密化]。TDMクロック10nsに対して0.0025%。PZT stretcher (ストローク±50μm, 分解能<1nm) による微調整で残差<0.1ps達成。** 理由: DNANF(0.013dB)は低損失だが偏波非保持。200m伝搬で~20°偏波回転が生じスクイージング-0.27dB劣化。PMF(0.07dB, 0.35dB/km)は偏波完璧に保持し追加損失+0.057dBのみ。コストも約90万円→約1.5万円で99%削減。
**PMF損失根拠**: Corning PM1550: 0.40-0.50dB/km、Fujikura PMJ-3AL: 0.30-0.40dB/km。設計値0.35dB/km (高グレード品選定)。200m×0.35=0.07dB。

### 3.4 ビームスプリッタ設計

| BS | 方式 | 損失 | 分配比精度 |
|----|------|------|-----------|
| BS₁ | ファイバカプラ (PMF) | 0.1dB | 50±0.5% |

## 4. ホモダイン測定系

### 4.1 構成

```
out_A ──────────────────────┐
                            ├──► [Balanced Homodyne Detector A]
LO (from master laser) ────┘           │
                                       ├── q_A or p_A (選択的)
                                       ▼
                                [Flash ADC 6bit 1.5GSPS + Pipeline 14bit 400MSPS]
                                       │
                                       ▼
                                [FPGA: 基底選択 + GKP decode]

out_B ──────────────────────┐
                            ├──► [Balanced Homodyne Detector B]
LO ────────────────────────┘           │
                                       ├── q_B or p_B
                                       ▼
                                [Flash ADC 6bit 1.5GSPS + Pipeline 14bit 400MSPS]
                                       │
                                       ▼
                                [FPGA: GKP soft info extraction]
```

### 4.2 ホモダイン検出器仕様

| パラメータ | 仕様 | 根拠 |
|-----------|------|------|
| PD | InGaAs balanced PD (Thorlabs PDB480C相当) | 量産品 |
| 量子効率 | ≥98% (目標) | カスタムPD必須 市販InGaAs PD QE≈95%。Phase -1でQE≥98%カスタムPD開発 |
| 帯域 | 1.6GHz | 100MHzクロックに十分 |
| CMR (コモンモード除去) | >35dB | バランス精度 |
| NEP | <5pW/√Hz | 量子雑音限界で動作 |
| LO power | 10mW/ch | ショットノイズ >> 電子ノイズ |
| チャネル数 | **2** (out_A + out_B) | macronode最小構成 |
| 動作温度 | **室温** | 冷却不要 |
| コスト | 約7万円/ch | テレコム量産品 |

### 4.3 測定基底の切替

CV測定型量子計算では、各モードの測定基底角度θを適応的に選択:
- θ = 0: q測定（X基底）
- θ = π/2: p測定（Z基底）
- θ = 任意角度: ユニバーサルゲート実行

**基底切替の実装**: LO位相をEOMで変調
- EOM応答時間: <1ns (LNOI EOM)
- 必要切替精度: ±1° → スクイージング劣化<0.1dB
- 切替速度: クロック速度100MHzで各モードごとに設定可能

## 5. 表面符号への埋め込み

### 5.1 Macronode → 表面符号マッピング

macronode latticeの各ノードにGKP qubitを配置し、表面符号の格子を構成:

```
表面符号 d=7 (49 data qubits + 48 ancilla qubits = 97 qubits)

macronode lattice上の配置:
  時間方向: 7 macronodes × 2 (data + ancilla) = 14 time steps / syndrome round
  列方向: 7 columns (空間方向をTDMで逐次処理)

  1 syndrome round = 14 time steps × 7 columns = 98 modes × 10ns = 980ns ≈ 1μs
  d syndrome rounds for full QEC cycle = 7 × 980ns = 6.86μs ≈ 7μs

  ※注: postselection (P_s≈10⁻³) 考慮後の実効レートは §5.2 参照
```

### 5.2 QEC速度

| パラメータ | 値 | 備考 |
|-----------|-----|------|
| 表面符号距離 d | 7 | |
| 1シンドロームラウンド | 980ns | 14 steps × 7 cols × 10ns |
| フルQECサイクル (d rounds) | **~7μs** | 7 × 980ns = 6.86μs |
| TDM直接レート (postselection無視) | ~143kHz | 1/7μs |
| **postselection込み有効レート** | | **以下参照** |

**ゲートレート定義 (P_s→P_round/p_acc記号分離)**:

**重要**: 旧P_s記号は廃止。p_acc=93%(単一モードacceptance rate)とP_round=p_acc^98≈10⁻³(ラウンド全モード同時高信頼)に分離。P_round=10⁻³は「1ラウンド内の全98モードが高信頼(δ<δ_max)で揃う確率」。
全モードは常にホモダイン測定され、soft-info重み付きデコーダに入力される（破棄なし）。

| 運用モード | ゲートレート計算 | d=7 値 | 備考 |
|-----------|----------------|--------|------|
| **全モード利用 (soft-info UF, ベースライン)** | 1 / T_QEC = 1 / 6.86μs | **~146kHz** | 全モードを信頼度重み付きで利用。p_L~10⁻⁵(現実Δ=0.12) |
| 高信頼モード待機 (strict postselection) | 686 modes / (100MHz×P_s) | **~146Hz** | 高信頼モード98個×7ラウンド分待機。p_L~4×10⁻⁷(Δ=0理想) |
| **WDM並列 (製品スペック)** | 146kHz × 7ch (全モード) / 146Hz × 7ch (strict) | **~1MHz (QECレート) / ~1kHz (strict論理ゲート)** | WDM 7ch並列。注: ~1MHzはQECサイクル処理レート、論理ゲートレート(strict)は~1kHz |

```
■ 全モード利用モード (ベースライン):
  全100MHzモードを測定、信頼度で重み付け → erasure+soft-info UF
  QECサイクル = 7 × 980ns = 6.86μs → 論理ゲートレート ~146kHz/ch
  p_L推定: ~10⁻⁵ (Δ=0.12現実)、erasure率7%は閾値24.9%の28%で余裕あり

■ strict postselectionモード (高品質):
  高信頼モード(per-mode 93%合格)が1ラウンド分揃うのを待つ
  P_s_round = 0.93^98 ≈ 10⁻³ → 有効高信頼ラウンドレート = (100MHz/98)×10⁻³ ≈ 1.02kHz
  1 QECサイクル = 7高信頼ラウンド → 待ち時間 7/1.02kHz ≈ 6.86ms → ~146Hz
  p_L推定: ~4×10⁻⁷ (Δ=0理想上限)
```

**全モード利用アーキテクチャ**:

TDMクラスタ状態は100MHzで連続生成され、**全モード**が破壊的にホモダイン測定される。
postselectionは「破棄」ではなく「信頼度分類」として機能:

```
[TDM 100MHz] → [全モード即時ホモダイン測定] → [古典データストリーム]
                                                      │
                                               [GKP格子マッピング + confidence計算]
                                                      │
                                          全モードのbit + confidence → soft-info UFデコーダ
                                                      │
                                          高信頼(93%): weight大、低信頼(7%): erasure相当weight
```

物理的仕組み:
1. macronode latticeは2D entanglementを「全モード」に対し生成済み
2. ホモダイン測定でCV entanglementが古典相関（シンドローム情報）に変換される
3. **全モード**のbit値+confidence値がデコーダに入力される（破棄なし）
4. 低confidence モード(7%)はweight小としてUFクラスタ成長で自然に扱われる
5. → 実効ゲートレート = 1/T_QEC ≈ 146kHz (P_s無関係)

**P_s=10⁻³の正確な意味**: 98モード全てが高信頼(δ<δ_max)で揃う確率 = 0.93^98 ≈ 10⁻³。
これは「strict postselection」モードで最高品質のQECを行う場合のみ関連。
通常運用（全モード利用）ではP_sはゲートレートに影響しない。

**Native Tゲート (蒸留不要)**: 論理ゲートレート = Tゲートレート

**注**: Executive Summary記載値 (Edu 10kHz, Pro 3kHz, Max 1kHz) は
WDM並列化 (5-20ch) による高速化を織り込んだターゲット仕様。
単一チャネルでの上記値が物理的下限。

## 6. モジュール物理配置 (ラック内)

```
1U OPAモジュール:  [OPA×8]  (§3, 02_opa-source.md)
    │ PMF×8
    ▼
```

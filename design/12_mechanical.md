**熱設計リスク注記 (v1.6追加)**:
FPGA VE2302 TDP=25W + DC-DC損失9W + ADC 10W = Zone C合計44W（+DAC 5W=49W）。
Cu HP ×3 (各50W容量) → HP合計150W容量で44W負荷 → 十分。
ただしHP熱抵抗: R_HP≈0.5K/W (sintered wick, φ6mm, 80mm) × 44W/3本 = 7.3K。
FPGA T_j = T_ambient(25℃) + ΔT_enclosure(15K) + ΔT_HP(7.3K) + ΔT_TIM(3K) = **50.3℃**。
VE2302 T_j,max = 100℃ (industrial grade)。マージン= 100 - 50.3 = **49.7℃** → 十分。
(注: 民生grade=85℃でもマージン34.7℃で十分。初期解析でT_j=87℃と見積もる場合は
3本HP並列の分散効果を見落としている。1本集中時: 44W×0.5=22K → T_j=65K。)
**Phase 0a検証**: 実機でFPGA junction温度をXSDBで読み取り、T_j<70℃を確認。
ambient 35℃環境でもT_j<60℃(35+15+7.3+3=60.3℃)と予測。

### 2.4 接続損失バジェット (ファイバ配線起因)

| 接続種別 | 箇所数 | 単位損失 | 合計 |
|---------|:---:|:---:|:---:|
| FC/APC コネクタ | 8 | 0.15dB | 1.2dB |
| 融着接続 | 12 | 0.02dB | 0.24dB |
| スプール入出力 | 2 | 0.01dB | 0.02dB |
| **合計** | | | **1.46dB** |

⚠️ **ノイズバジェットの1.29dB(v1.2)中、光学損失許容1.22dBを超過！**

**対策**: FC/APCコネクタを全て融着接続に変更
- 変更後: 融着20箇所 × 0.02dB = 0.40dB + スプール0.02dB = **0.42dB**
- ノイズバジェット内に収まる
- トレードオフ: 保守時に再融着が必要 → **OPAモジュールはコネクタ式を維持 (交換可能にするため)**

**最終配線損失**:
- OPA-EOM: FC/APC ×2 = 0.30dB
- その他: 融着 ×16 = 0.32dB
- スプール: 0.02dB
- **合計: 0.64dB** (ノイズバジェットの「OPA結合0.3dB + PMF伝搬0.01dB + BS 0.3dB」に含まれる)

## 2.5 製造キャリブレーション手順 (v1.6追加)

各ユニットは固有のファイバパス損失・位相特性を持つため、per-unit較正が必須。

**較正シーケンス (合計~4h、Phase 1自動化目標2h)**:

| # | 手順 | 時間 | 合否基準 | 不合格時 |
|---|------|:----:|---------|---------|
| 1 | OPA CWスクイージング確認 | 30min | σ_gen ≥ 12.5dB (spec -0.5dB) | OPAモジュール交換 |
| 2 | ホモダインバランス調整 | 20min | CMR ≥ 35dB (100MHz) | PD再アライメント |
| 3 | PLL 3層ロック取得+安定性 | 45min | ±0.5° RSS, 30min持続 | PZT/EOM調整 |
| 4 | FF-1レイテンシ測定 | 15min | ADC trigger→DAC valid ≤ 27ns (設計ベースライン) | ケーブル長調整 |
| 5 | σ_eff統合測定 (AWG込み) | 30min | ≥ 10.5dB (shipping spec) [v3.1: 設計公称値に対し0.4dBマージン] | 損失源特定→再スプライス |
| 6 | GKP postselection動作確認 | 30min | P_mode ≥ 90% (δ=0.19√π) | δ_max再設定 |
| 7 | 温度サイクル安定性 (25→35→25℃) | 60min | σ_eff変動 < 0.3dB | Peltier PID再調整 |
| 8 | 48hバーンイン (無人) | (48h) | エラーなし、σ_eff drift <0.1dB | 初期不良排除 |

**自動化計画**: Phase 1でステップ1-6をFPGA自己診断ルーチン化。
ユーザー向け「自動キャリブレーション」(10_portable.md §4.5 Phase 7-8)はこの製造較正のサブセット。

**較正データ記録**: 各ユニットに固有の較正証書（σ_eff実測値、PLL BW、各チャネル損失）を添付。
シリアル番号+較正日+有効期限(1年)をFPGA不揮発メモリに書込み。

|---------|--------|--------|------|

| PZT driver | DC-10kHz | 3W | C |
| Master laser TEC | DC | 5W | A |

### 3.2 被害系の特定

| 被害系 | 感度 | 影響 |
|--------|------|------|
| Balanced PD (ホモダイン) | ~pW | **最重要**: 量子ノイズ限界で動作 |
| OPA出力光 | 位相 ±0.05° | 位相ノイズ → スクイージング劣化 |
| ADC入力 | 14bit LSB | 量子化精度に影響 |

### 3.3 EMC対策

```
┌──────── アルミ筐体 (シールド筐体、連続導通) ────────┐
│                                                     │
│  ┌── Zone A (光学) ─┐  ┌── Zone B ──┐  ┌── Zone C (電子) ──┐
│  │  銅箔シールド壁   │  │            │  │  銅箔シールド壁   │
│  │  (0.1mm Cu)     │  │  ファイバ   │  │  (0.1mm Cu)     │
│  │                  │  │  のみ      │  │                  │
│  │  PD, OPA,       │  │  (EMI透過  │  │  FPGA, ADC,     │
│  │  Laser          │  │   ゼロ)    │  │  DC-DC          │
│  └──────────────────┘  └────────────┘  └──────────────────┘
│       ↑                     ↑                ↑
│  アナログGND              光ファイバ         デジタルGND
│  (分離)               (電気的に絶縁)         (分離)
└────────────────────────────────────────────────────┘
        └─── 1点接地 (背面GND端子) ───┘
```

| 対策 | 詳細 | 効果 |
|------|------|------|
| **ゾーン間シールド** | Zone A/C間に銅箔シールド壁 (0.1mm) | -60dB@1GHz |
| **光ファイバ絶縁** | Zone A-C間は全てファイバ接続 (電気配線ゼロ) | 完全電気絶縁 |
| **GND分離** | アナログGND / デジタルGND を分離、1点接地 | GNDループ排除 |
| **DC-DCシールド** | スイッチング電源を個別シールド缶内に格納 | -40dB@2MHz |
| **ADCガード** | ADC入力にコモンモードチョーク + ディファレンシャルフィルタ | -30dB |
| **EO driver配線** | ツイストペア + シールド (SMA同軸) | -40dB |
| **筐体シールド** | アルミ筐体の全接合面に導電ガスケット | -80dB@1GHz |

### 3.4 EMC規格適合

| 規格 | 要件 | 対策 |
|------|------|------|
| CISPR 32 Class B (放射) | <40dBμV/m@10m | シールド筐体 + フィルタ |
| CISPR 32 Class B (伝導) | <60dBμV@AC線 | ACフィルタ (外付けACアダプタに内蔵) |
| IEC 61000-4-2 (ESD) | ±8kV接触 | GND端子 + 筐体アース |
| IEC 61000-4-3 (放射イミュニティ) | 3V/m | 光ファイバ絶縁で本質的に免疫 |

**重要**: Zone A-C間が光ファイバのみで接続されるため、EMCは本質的に有利。
電気的クロストーク経路が物理的に存在しない。

### 4.3 重要なDFM設計判断

| # | 設計判断 | 理由 | トレードオフ |
|---|---------|------|------------|
| 1 | **筐体一体成型** (上蓋+本体の2ピース) | 部品点数削減、シールド性 | CNC加工コスト高 |
| 2 | **OPAモジュール交換式** | OPA故障時のフィールド交換 | FC/APCコネクタ ×2 (損失+0.3dB) |
| 3 | **ファイバ融着** (OPA接続以外) | 損失最小化 (0.02dB/点) | 保守時に再融着必要 |
| 4 | **FPGA/ADCスタック** | フットプリント削減 | 放熱に銅ヒートパイプ必要 |
| 5 | **PMFスプール交換不可** | 融着で固定 (損失最小) | d変更には筐体ごと変更 |
| 6 | **WDM AWGドーターカード式** | Phase 0→Phase 2+のch数アップグレード | コネクタ損失 |

## 議題1: なぜシミュレーションが設計文書より440倍良いのか

### 事実

| Phase | 設計文書 p_L(d=7) | Exp-A2 (code-capacity) | 比率 |
|-------|-----------------|----------------------|------|
| Phase 1 (8.5dB) | 4.4×10⁻³ | <10⁻⁵ | >440× |
| Phase 2+ Real (9.3dB) | 3.3×10⁻⁴ | <5×10⁻⁶ | >66× |

### 原因分析

**設計文書の計算方法**:
```
p_L = A × (p_phys/p_th)^((d+1)/2)
A = 0.03, p_th = 1.5% (Noh & Chamberland 2022, circuit-level)
Phase 1: p_phys/p_th = 0.62, p_L(d=7) = 0.03 × 0.62⁴ = 4.4×10⁻³
```

**Exp-A2の結果**:
```
Phase 1 フィット: A = 0.070, p_phys/p_th = 0.089, Λ = 11.2
→ p_th(code-capacity, soft-info) ≈ 10.4%
→ p_L(d=7) = 0.070 × 0.089⁴ ≈ 4.4×10⁻⁶
```

**差の本質**: code-capacity閾値(~10%) vs circuit-level閾値(~1.5%) の差。
p/p_th が 0.62 → 0.089 になることで、(d+1)/2=4乗で (0.62/0.089)⁴ ≈ 2,300倍の差が生まれる。

### 議論ポイント

**A(アーキテクト)**: 設計文書はcircuit-levelの保守値を使っている。これは安全マージンとして正しい。しかし、CV方式の物理的特性（ホモダインが常に成功）を考えると、実際の性能はcode-capacityとcircuit-levelの間にある。

**B(QECシミュレータ)**: Exp-A3（現在実行中）で phenomenological モデルの結果を確認する。CV方式ではp_meas = p_dataなので、phenomenologicalがCV方式の正しいモデルに近い。

**C(光学モデラー)**: code-capacityとcircuit-levelの差は「測定ノイズ」の有無。CV homodyne では追加の測定ノイズはないが、フィードフォワード遅延によるモード間相関がある。これはどちらのモデルにも含まれていない。

**D(デコーダ研究者)**: soft-info MWPM はcode-capacityで最大の効果を発揮する。phenomenologicalでは時間方向のマッチングが加わり、soft-infoの効果が希釈される可能性がある。

### 合意事項案

1. 設計文書の保守的予測 (p_L = 3.3×10⁻⁴ at Phase 2+) は**下界保証**として維持
2. code-capacity結果は**楽観的上界**として位置づけ
3. Exp-A3 (phenomenological) の結果で**現実的予測値**を更新
4. 論文では3モデルすべての結果を報告し、CV方式の物理的特性を議論

## 議題3: 命題P2「Scalable architectureが存在する」の証明戦略

### TDM macronodeのスケーラビリティ

Exp-A2で d=3,5,7,9 のすべてで動作を確認。TDMアーキテクチャの利点:

```
論理qubit数 = N_row / (2d²)

d=7の場合:
  1 qubit:    98モード × 10ns = 980ns
  10 qubits:  980モード = 9.8μs
  100 qubits: 9,800モード = 98μs
  1000 qubits: 98,000モード = 980μs ≈ 1ms

→ ハードウェア変更なしで1000 qubitまで逐次スケール
→ WDM並列化で同時qubit数も拡張可能
```

### 証明に必要な追加要素

1. **大規模回路のコンパイル**: TDM上での論理ゲート(CNOT, T-gate)の実行シミュレーション
2. **マジック状態蒸留**: CV方式でのマジック状態蒸留プロトコルの確認
3. **リソースオーバーヘッド**: 既存方式（超伝導、イオントラップ）との比較

### 議論ポイント

**A**: TDMの逐次スケーリングは原理的に証明済み。しかし、1000 qubit回路の実行時間が~1msで実用的かは応用依存。量子化学(VQE)なら十分だが、暗号解読(Shor)では不足の可能性。

**C**: WDM並列化（Phase 3+で20ch）で実質的な同時qubit数を拡張できる。これは室温動作の利点（冷凍機のスケーリング制約がない）と直結。

## 議題4: 命題P3「Cryogenic infrastructure不要」の物理的根拠

### CV方式が室温で動作する3つの物理的理由

**理由1: 光子エネルギー >> kT**
```
1550nm光子: E = hc/λ = 0.80 eV = 9,280 K (等価温度)
室温 kT: 0.026 eV = 300 K

E/kT = 31 → 熱光子数 n_th = 1/(e^31 - 1) ≈ 3×10⁻¹⁴
→ 室温での熱ノイズは量子ノイズの10⁻¹⁴倍 → 完全に無視可能
```

**理由2: ホモダイン検出は室温で量子限界動作**
- InGaAs PD (室温): QE ≥ 98%, NEP < 5pW/√Hz
- LO 10mW: ショットノイズ >> 電子ノイズ
- SNSPD不要（単一光子検出が不要）

**理由3: スクイーズド光の生成は室温プロセス**
- PPLN導波路OPA: 室温動作（温度安定化 ±0.01℃のみ必要）
- 非線形結晶の位相整合は温度チューニングで実現
- 冷凍機は不要

### DV方式が冷凍機を必要とする理由（比較）

```
DV方式 (FBQC, Xanadu/PsiQuantum):
  - SNSPD動作温度: 0.8-4K → 冷凍機必須
  - 量子ドット: 4-10K → 冷凍機必須
  - 単一光子の熱ノイズ耐性: 低い → 冷凍機必須

CV方式 (TAROS):
  - ホモダイン: 室温動作
  - OPA: 室温動作
  - スクイーズド光の熱ノイズ耐性: 完全（上記理由1）
```

### 室温動作の唯一のリスク

- **位相ドリフト**: 室温での熱膨張によるファイバ長変動 → PLL (BW ≥ 500kHz) で補償
- **機械振動**: 光学台の振動 → パッシブ防振 + アクティブPLL
- **温度安定化**: OPA位相整合の温度感度 → ±0.01℃ TEC制御

これらはすべて**エンジニアリング課題**であり、**物理的限界**ではない。

## 議題5: 研究の新規性 — 何が世界初か

### Exp-A2で証明できたこと（暫定）

1. **室温CVフォトニックアーキテクチャでのQECスケーリングの数値的実証**
   - d=3→5→7→9での指数的誤り抑制
   - soft-info MWPM による 5-9× 改善
   - TAROSの設計パラメータでの独立検証

2. **Code-capacity閾値の定量化**: p_th ≈ 10% (soft-info MWPM, GKP表面符号)

3. **3モデル比較** (Exp-A3完了後): code-capacity vs phenomenological vs circuit-level で、CV方式の物理的特性を定量化

### 先行研究との差別化

| 先行研究 | 内容 | 本研究との差 |
|---------|------|------------|
| Noh & Chamberland 2022 | GKP soft-info MWPM | 閾値計算のみ。距離スケーリングのシステム設計検証なし |
| Stafford+ 2025 | Macronode TDM閾値 | 閾値のみ。具体的なハードウェアパラメータでのp_L計算なし |
| Bourassa+ 2021 (Xanadu) | Photonic FTQC blueprint | DV方式+冷凍機前提。室温CV方式の検証なし |
| Fukui+ 2018 | GKP+表面符号 | 一般的な閾値解析。macronode TDMアーキテクチャ未検討 |

**本研究の独自性**: 具体的な室温ハードウェア設計 (PPLN OPA, PMF, InGaAs PD) のパラメータで、macronode TDM上のGKP表面符号QECスケーリングを数値的に実証した初の研究。

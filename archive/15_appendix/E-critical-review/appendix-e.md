# Appendix E: v2.0 Critical Review 完全記録

本Appendixは、v2.0設計書に対して6名の専門家チーム（量子物理学者、QEC理論家、フォトニクスエンジニア、システムアーキテクト、数値整合性監査人、論文検証スペシャリスト）が実施したCritical Reviewの完全記録である。v3.0の設計変更（セクション1-14、Appendix D）の根拠となった全指摘事項、専門家間の議論、および改善勧告を収録する。

## E.1 レビュー概要

**対象文書**: 小型フォトニック量子コンピュータ_理論設計書_v2.0.pdf (26p)
**レビュー日**: 2026-05-04
**検出問題数**: 致命的 15件、重大 20件、中程度 16件、軽微 11件

v2.0はv1.0の13件のバグを修正した誠実な改訂であるが、新アーキテクチャの採用に伴い新たな致命的矛盾が複数発生していた。特に設計の第四原則「数値整合性を全文書範囲で保証する」が守られていなかった。

## E.2 致命的問題 TOP 10

### E.2.1 FATAL-1: ファイバ遅延損失の桁違い矛盾

**検出者**: 量子物理学者、フォトニクスエンジニア、数値監査人（3名が独立に検出）

v2.0 SS7.1には「ファイバ遅延50kmファイバ: 0.025%」、SS6.3.2には「0.2 dB/km x 50m = 0.01 dB/loop, 総損失 0.8 dB」と記載されていた。0.8 dBは16.8%損失であり、記載の0.025%と672倍の乖離がある。「50km」の表記も誤記（実際は50m x 80 = 4km）であった。

Phase 1総損失は記載の11.9%ではなく実際には約28%となり、楽観閾値9%を3倍超過する。この1件のみで損失予算全体が破綻する。

**v3.0での対応**: HCF 600m単一パス方式に変更（セクション3.4、6.3.2）。損失0.07dB。

### E.2.2 FATAL-2: SNSPD周波数多重の原理的誤解

**検出者**: フォトニクスエンジニア

SNSPDは波長非選択的検出器であり、光子エネルギー情報は吸収時に失われる。v2.0の「256ch空間 + 16ch周波数多重 = 実効4,096ch」は、AWGで空間分離後に4,096個の物理SNSPDが必要であることを意味する。256個のSNSPDで4,096chの情報を得ることは物理的に不可能であった。

**v3.0での対応**: SNSPD 8,192個 + 行列読み出しに変更（セクション6.4）。

### E.2.3 FATAL-3: QD 1550nm帯の要求スペック同時達成が未実証

**検出者**: 量子物理学者、リスクアナリスト

g^2(0)<0.01 + HOM可視度99.5% + 抽出効率95%を1550nm帯で同時達成した報告は存在しなかった。現状のInAs/InP QDベストはg^2(0)約0.02-0.05、HOM約90-95%であり、99.5% HOMにはT2*>200nsが必要だがInAs/InP系の報告値はT2*約数ns-数十nsであった。

**v3.0での対応**: 925nm GaAs QD + LNOI QFCに変更（セクション4.2、4.3）。

### E.2.4 FATAL-4: SHYPS符号のSingle-shot性が未証明

**検出者**: QEC理論家

Single-shot QECの必要条件（confinement property, Bombin 2015）をSHYPS符号が満たす証明・数値検証が設計書に一切なかった。Hypergraph product符号は一般にsingle-shot性を持つとは限らない。不成立の場合、d=8ラウンドのシンドローム測定が必要となり、サイクル時間が8倍（20us→160us）に増大して性能予測全体が崩壊する。

**v3.0での対応**: 3パス並行設計を採用（セクション5.1）。confinement property検証をPhase -1の最優先タスクとして設定。

### E.2.5 FATAL-5: 閾値の乗算合成が理論的に不正当

**検出者**: QEC理論家

v2.0では0.79% → x7.2(adaptive bias) → 5.7% → x1.5(SHYPS) → 8.55% → x1.06(AlphaQubit) → 9.06%と計算していた。しかし、閾値は乗算で合成できる量ではなく、符号・デコーダ・ノイズモデルの三つ組に対して定まる非線形な臨界現象パラメータである。

**v3.0での対応**: 乗算合成を廃止し、Stimフレームワークによる直接シミュレーションに移行（セクション5.6）。

### E.2.6 FATAL-6: BaTiO3 240個のSiN集積プロセスが存在しない

**検出者**: フォトニクスエンジニア

BaTiO3 on SiNの300mmウェハプロセスは研究段階で単一デバイスレベルの実証のみであった。240個のアレイ集積（均一な結晶配向、消光比30dB）を達成する技術は2026年時点で未存在であった。

**v3.0での対応**: LNOIに変更（セクション6.2.1）。商用MPWサービスが利用可能。

### E.2.7 FATAL-7: コスト27億円 vs 33億円の不一致

**検出者**: 数値監査人、リスクアナリスト

表紙には「ハードウェア概算27億円」、SS11 BoMにはハードウェア29.22億円 + ソフト・人件費4億円 = 約33億円、Appendix Cには「33億円」と記載されていた。6億円（18%）の乖離は投資家向け資料として致命的な不整合であった。

**v3.0での対応**: ゼロベース再積算により70-90億円と再見積り（セクション11）。

### E.2.8 FATAL-8: 異なる符号間のp_L外挿が不当

**検出者**: QEC理論家、論文検証スペシャリスト

BB Gross [[144,12,12]] d=12のシミュレーション値（Bravyi 2024）を、SHYPS [[400,25,8]] d=8に「Malcolm 2025推定」として外挿していた。BB GrossとSHYPSは異なる符号族であり、距離・構造・デコーダ性能が全て異なる。

**v3.0での対応**: 各符号個別にStimシミュレーションで閾値を計算する方針に変更（セクション5.6.2）。

### E.2.9 FATAL-9: 光子効率パイプライン総合約1%

**検出者**: フォトニクスエンジニア

各段階の損失を正直に積算すると、QD発光95% x ファイバ結合90% x チップ結合79% x チップ内損失40% x ファイバ遅延5% x SNSPD 97% = 総合効率約1.3%（ファイバ遅延損失-13dBと仮定）であった。量子誤り訂正の閾値（損失<数dB）を大幅に超過し、計算が成立しないレベルの問題であった。

**v3.0での対応**: QFC + HCF + erasure管理による損失バジェットの全面再構築（セクション4.4、7.1）。

### E.2.10 FATAL-10: 物理誤り率pの値がセクション間で不統一

**検出者**: システムアーキテクト

SS2.3ではp = 5x10^-3を使ってp_L約10^-5を導出し、SS8.2では「物理融合誤り率: p = 10^-3」と記載していた。5倍の差があり、どちらのpで設計するかで符号設計・ハードウェア要件が根本的に変わる。

**v3.0での対応**: 階層的物理誤り率定義（p_erasure、p_depolarizing）を導入し統一（セクション2.3.1）。

## E.4 専門家クロス議論記録

### E.4.1 議論1: ファイバ遅延問題は設計を根本から無効化するか

**量子物理学者**: 50m x 80ループのファイバ遅延は合計4kmで、0.8dB損失と自分たちで書いておきながら、損失予算では0.025%としている。672倍の矛盾であり、これ1つで設計が成立しない。

**フォトニクスエンジニア**: さらに、0.8dBはファイバの伝搬損失のみ。80ループの入出力結合損失（スプライス0.02-0.05dB/点 x 160点 = 3.2-8.0dB）を加えると実効損失は4-9dB。全体で5-10dBは光子の70-90%が失われることを意味する。

**システムアーキテクト**: 時間多重方式では各光子が80ループ全てを通過するわけではない可能性がある。光子がN番目のタイムビンに配置される場合、N回のループ通過で済む。平均通過回数が40ループなら損失は半分。

**量子物理学者**: その場合でも、40ループで0.4dB + 結合損失1.6-4.0dBで計2-4.4dBの損失。損失予算には「0.025%」ではなく「30-60%」と書くべき。

**QEC理論家**: ファイバ遅延損失が仮に30%でも、FBQC融合の実効光子損失として閾値9%を大幅に超過する。ファイバ遅延ループ方式自体がこのアーキテクチャに適さない可能性がある。光メモリやマルチプレックス生成への転換が必要。

**数値監査人**: 根本的に、SS6.3.2の「総損失0.8dB」とSS7.1の「0.025%」が同じ文書内に存在することが第四原則違反。少なくとも一方を修正すべき。

> **合意**: ファイバ遅延の損失は設計の存亡に関わる致命的問題。光メモリや低損失オンチップ遅延への代替を検討すべき。

### E.4.2 議論2: SNSPD「256ch + 周波数多重16x = 4,096ch」は成立するか

**フォトニクスエンジニア**: SNSPDは周波数分解能を持たない。AWGで16chに空間分離した後、各々に独立SNSPDが必要。つまり物理的に4,096個のSNSPDが必要。これを「256ch + 16x多重」と表記するのは原理的な誤解。

**量子物理学者**: Transition-Edge Sensor (TES) なら光子数分解能がある。しかしTESは応答速度が遅く（約us）、50MHzの繰り返しには対応できない。

**システムアーキテクト**: 4,096個のSNSPDが必要なら、クライオ配線は4,096本に増加。JT 5W@4Kでは冷却能力が全く足りない。これは検出系だけでなくシステム全体の規模・コスト・消費電力を根本的に変える。

**リスクアナリスト**: BoMでSNSPD 256ch = 3億円と見積もっているが、4,096個必要なら48億円。これだけで総予算33億円を超過する。

**フォトニクスエンジニア**: 代替案として、SNSPDのrow-column multiplexingで物理デバイス数を減らすアプローチがあるが、読み出し帯域とcrosstalkの制約がある。

> **合意**: 4,096個のSNSPDが本当に必要かを明確化し、必要な場合はクライオ・コスト・電力を全面的に再計算すべき。

### E.4.3 議論3: SHYPS符号は本当にこの設計の基盤として適切か

**QEC理論家**: 3つの致命的問題がある。(A) Single-shot未証明、(B) 閾値乗算不当、(C) BB→SHYPS外挿不当。どれか1つでもSHYPS採用の根拠を崩す。

**論文検証スペシャリスト**: さらに、SHYPS符号は未査読プレプリント1本(Malcolm 2025, arXiv:2502.07150)にのみ依存している。Photonic Inc.の自社技術であり、独立した追試・検証が一切ない。

**リスクアナリスト**: BB符号へのフォールバックは「10倍リソース」と記載されているが、これは100論理qubitに1,000物理qubit相当が必要ということ。コスト33億円では収まらず、事実上「プロジェクト再設計」。

**QEC理論家**: kd^2/nの指標で見ると、BB Gross(12)がSHYPS(4)の3倍優れている。k/nだけの比較は距離d=12→8の劣化を隠蔽している。

**システムアーキテクト**: 仮にSHYPSが論文通りの性能を出したとしても、4ブロックx25 qubitの接続トポロジーが未定義。ブロック間CNOTが全CNOTの75%を占めるアルゴリズムでは、実効サイクル時間が大幅に増大する。

**量子物理学者**: そもそもSHYPSのFBQC上での損失閾値は、表面符号の閾値とは独立に計算されるべき。5.7% x 1.5 = 8.55%という計算には物理的根拠がない。

> **合意**: SHYPS符号の採用はPhase 0のフィージビリティスタディで検証すべき最優先事項。BB符号での設計を並行して維持すべき。

### E.4.4 議論4:「既知技術のみ」原則は守られているか

**リスクアナリスト**: 設計書は「既知技術の組合せのみを使い、未発見の物理現象に依拠しない」と宣言しているが、以下は全て未実証: 1550nm QD単一光子源（抽出効率99%）、SHYPS符号の実機動作、12 QD並列の位相同期、256ch SNSPD統合、BaTiO3 240個オンチップ集積、システム全体の損失5.3%。

**量子物理学者**: 各コンポーネントが個別に論文で報告されていることを「既知」と定義しているが、システム統合は未知の領域。「車輪とエンジンは既知技術だから自動車は既知技術で作れる」という論法と同じ。

**フォトニクスエンジニア**: 実際、BaTiO3のSiN集積は「単一デバイスの実験室実証」段階。240個アレイを量産レベルで作る技術は存在しない。これは「既知技術」ではなく「未来技術」。

**論文検証スペシャリスト**: さらに、設計パラメータの根拠となる文献のうち少なくとも4本が実在確認困難。未査読プレプリントへの依存度も高い。

**QEC理論家**: AlphaQubitの30%改善をsuperconducting surface code → photonic qLDPCに外挿しているのも「未知技術」の域。

> **合意**: 「既知技術のみ」原則は事実上守られていない。「2027年までに実証が必要な技術一覧」を正直に明示すべき。

### E.4.5 議論5: この設計は投資に値するか

**リスクアナリスト**: 3つの核心仮定の成功確率を各70%（楽観的）としても、全成功確率は34%。50%なら12.5%。33億円投資の期待値としては厳しい。

**フォトニクスエンジニア**: 技術的には致命的問題が6件ある。いずれか1つでもシステム全体を阻む。確率は独立ではなく相関するため、実際のリスクはさらに高い。

**QEC理論家**: ただし、設計書の「正直な但し書き」は誠実。Phase 1をフィージビリティスタディと位置づけ、Go/No-Go判定を設けている設計思想自体は正しい。

**論文検証スペシャリスト**: 学術的には、FBQCアーキテクチャの小型化設計としての価値はある。ただし「商用量子コンピュータの設計書」としてではなく「研究提案書」として評価すべき段階。

**システムアーキテクト**: Phase 1の目標を「損失<12%, p_L<10^-3」と設定しているが、本レビューの指摘通りなら損失12%すら達成困難。Phase 1の目標自体を再定義する必要がある。

**量子物理学者**: 推奨は、33億円の全投資の前に、1-2億円のフィージビリティスタディを6ヶ月実施すること。検証対象: (a) 1550nm QDの調達可能性、(b) SHYPSのStimシミュレーション検証、(c) ファイバ遅延の損失実測。

> **合意**: 現段階では「研究提案」として価値があるが、「投資可能な設計書」としてはFATAL問題の解決が先決。Phase 0の前に集中的予備検証を推奨。

## E.5 改善議論記録

レビュー完了後、同じ6名の専門家チーム（Dr. 田中（量子物理学者）、Dr. 鈴木（QEC理論家）、Dr. 山本（フォトニクスエンジニア）、Dr. 佐藤（システムアーキテクト）、Dr. 高橋（数値監査人）、Dr. 渡辺（論文検証スペシャリスト））が5テーマの改善議論を実施した。

### E.5.1 テーマ1: ファイバ遅延問題の根本的解決策

**問題**: 80段ファイバ遅延ループ（4km）の損失が98.7%でシステム不成立。

**議論要旨**:
- 結合損失0.1dB x 160回 = 16dBであり、ファイバ遅延ループ方式は原理的に不成立
- パウリフレーム追跡を完全実装すれば、リアルタイム遅延要件を20us→2usに短縮可能。遅延が必要なのはマジック状態蒸留時のみ
- ホローコアファイバ（0.11dB/km、NKT Photonics/Lumenisity）で600m遅延なら損失0.2dB
- Boosted fusion（Gimeno-Segovia 2015）で損失閾値を2.7%→6%に緩和可能

**合意事項**:
- 80段ファイバ遅延ループは廃止
- パウリフレーム追跡で遅延を2us以下に短縮
- Boosted fusionで損失閾値を約6%に緩和
- ホローコアファイバ600mで損失0.2dB（閾値内）
- 空間多重主体のアーキテクチャに移行

### E.5.2 テーマ2: 検出系の再設計

**問題**: SNSPDの波長多重は物理的に不可能。実際には4,096個の物理デバイスが必要。

**議論要旨**:
- SNSPDは広帯域吸収体であり波長識別は原理的に不可能
- PNR不要（0/1閾値検出で十分）。時間多重深さ4で必要SNSPD数は約8,192個
- 行列読み出し（91x91配列、182本配線）で配線問題を解決
- 検出損失はerasureとして管理（閾値約50%に対して十分マージンあり）
- クロストーク約1%がdepolarizing閾値とほぼ同値であり最大リスク

**合意事項**:
- SNSPD 8,192個（時間多重深さ4との組み合わせ）
- Fiber-coupled SNSPD + 行列読み出し（182配線、熱負荷1.8mW@4K）
- 検出損失はerasureとして管理（閾値50%に対して十分）
- クロストーク約1%が最大リスク。短期は個別読み出し、中期でクライオ内ASIC開発
- コスト: SNSPD系全体で約7億円

### E.5.3 テーマ3: SHYPS符号問題の解決と代替案

**問題**: Single-shot性未証明、閾値乗算不当、未査読プレプリント1本への一極依存。

**議論要旨**:
- 4つの致命的問題を分解: (A) Single-shot未証明、(B) 閾値乗算不当、(C) BB→SHYPS外挿不当、(D) 一極依存
- 代替候補として4D Hypergraph Product Code (HPC)が最有力。Single-shot性証明済み、4D接続性が光子FBQCと高親和性
- 光子FBQCでは次元制約がない（任意の光子ペアを融合可能）→ 4D HPCの物理的実装障壁が低い
- 3パス並行設計: Primary=SHYPS、Secondary=4D HPC、Tertiary=表面符号

**合意事項**:
- 閾値の乗算合成は廃止。Stimフレームワークで統一ノイズモデル閾値を再計算
- SHYPSのconfinement property数値検証を最優先タスクとして実行
- 3パス並行設計: SHYPS / 4D HPC / 表面符号
- ゲート判断: 2027年Q2にSHYPS検証、Q4にHPCシミュレーション
- 表面符号フォールバックはリソース16倍で事実上のプロジェクト中止判断

### E.5.4 テーマ4: QD光源と物理層の再構成

**問題**: 1550nm QDスペック未実証、BaTiO3集積未存在、物理誤り率不統一。

**議論要旨**:
- 1550nm InAs/InPでの3要件同時達成は5-10年不可能。925nm GaAs QD + QFCに変更を推奨
- QFC: LNOI薄膜導波路で効率87%（損失0.6dB）。BaTiO3→LNOIに変更（商用MPWで即利用可能）
- 物理誤り率を階層定義で統一: p_erasure約40-45%（閾値50%内）、p_depolarizing約0.7-1.0%（閾値1%ギリギリ）
- depolarizingマージンの薄さが全設計の最大リスク。QD品質改善（HOM>99%）が鍵

**合意事項**:
- 1550nm InP QD → 925nm GaAs QD + LNOI QFCに変更
- BaTiO3 → LNOIに変更（商用利用可能）
- 物理誤り率統一: p_erasure約40-45%, p_depolarizing約0.7-1.0%
- depolarizingマージンの薄さが全設計最大のリスク
- コスト影響: ネット約+7,000万円

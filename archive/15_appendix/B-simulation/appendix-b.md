# Appendix B: シミュレーションプロトコル（改訂）

## B.1 Stimフレームワークによる閾値計算手順

### B.1.1 概要

Stim [Gidney2021] はGoogle開発のスタビライザ回路シミュレータで、大規模な量子誤り訂正回路のノイズ下性能を高速に評価できる。本設計ではPhase -1の主要解析ツールとして使用する。

### B.1.2 手順

```
Step 1: 回路定義
  - 3符号（SHYPS, 4D HPC, 表面符号）のsyndrome extraction回路を
    Stimの回路記述言語で定義
  - デュアルレール符号化を考慮し、各物理操作を
    fusion測定 + Pauli feedback のシーケンスとして記述

Step 2: ノイズモデル定義
  - erasure_error(p_e): 各光子に対し確率p_eでerasureが発生
  - depolarize(p_d): fusion操作後に確率p_dでdepolarizing errorが発生
  - measurement_error(p_m): 検出器の暗計数に起因する測定エラー（p_m < 0.001）

Step 3: パラメータスイープ
  - p_e: 0.1 から 0.9 まで 0.02刻み（41点）
  - p_d: 0.001 から 0.05 まで 0.002刻み（25点）
  - 符号距離: d = 4, 6, 8, 10, 12
  - 各パラメータ点で 10⁶ shot のMonte Carlo
  - 合計: 41 × 25 × 5 × 3符号 = 15,375 シミュレーション点

Step 4: 閾値抽出
  - 各(p_e, p_d)に対し、d依存の論理エラー率 p_L(d) を取得
  - p_L(d₁) = p_L(d₂) となる交差点を閾値とする
  - 2D閾値マップ: p_e vs p_d の平面上に閾値曲線を描画

Step 5: 動作点評価
  - 本設計の推定動作点 (p_e ≈ 0.85, p_d ≈ 0.5-1.0%) を
    閾値マップ上にプロットし、マージンを定量評価
```

### B.1.3 計算リソース

```
1シミュレーション点: ~10秒（10⁶ shots、d=8、Stim最適化済み）
全15,375点: ~42.7時間
GPU並列化（H200）: 8並列で ~5.3時間
Phase -1での全シミュレーション（パラメータ変更含め3周）: ~16時間
```

### B.1.4 出力物

1. 3符号の2D閾値マップ（p_e - p_d平面）
2. 各符号の最適動作領域の特定
3. 本設計パラメータでの論理エラー率予測値
4. 符号選択の定量的推奨

## B.2 QD光源シミュレーション

### B.2.1 目的

12個のGaAs QDが生成する光子の品質（純度、不可弁別性）を予測し、fusion操作のdepolarizing率への影響を定量化する。

### B.2.2 モデル

```
QD発光の量子光学モデル:
  Jaynes-Cummingsハミルトニアン（QD-cavity結合）
  H = ℏω_c â†â + ℏω_QD σ†σ + ℏg(â†σ + â σ†)

  Purcell効率: F_P = 4g²/(κγ)  （>10 で高効率収集）
  g: QD-cavity結合強度 (~50 μeV)
  κ: cavity減衰率 (~200 μeV)
  γ: QD自然放出率 (~1 μeV)
  → F_P ≈ 50

不可弁別性のモデル:
  HOM visibility V = 1 - (γ*/γ_total) - (1/2)(κ_phonon/γ_total)
  γ*: 純粋位相緩和率（~0.5 μeV @ 4K）
  κ_phonon: フォノンサイドバンド寄与（~0.1 μeV @ 4K）
  γ_total = γ + γ* + ... ≈ 1.6 μeV
  → V ≈ 1 - 0.31 - 0.03 ≈ 0.97（測定値と整合）
```

### B.2.3 QFC後の品質

```
QFC変換による品質劣化:
  HOM visibility (after QFC) = V_QD × (1 - p_noise)
  p_noise: QFCノイズ光子率 (< 0.001)
  → V_after_QFC > 0.97 × 0.999 = 0.969

12個のQD間の不可弁別性:
  異なるQD間のHOM: V_inter ≈ V_QD × 波長重なり積分
  波長重なり: QDの均一性に依存。温度チューニングで ±0.5nm以内に調整
  → V_inter > 0.95（Phase 0で実測必須）
```

### B.2.4 シミュレーション手順

```
1. 12個のQDパラメータ（ω_QD, g, γ*, T_phonon）のランダム生成
   → 正規分布、μとσは文献値 [Tomm2021] に基づく
2. 各QDペアのHOM visibilityを計算
3. QFC変換後のvisibilityを算出
4. fusion操作のシミュレーション（density matrix形式）
5. depolarizing率 p_d の抽出
6. 1000回の統計サンプリングで分布を取得
```

## B.3 全システムMonte Carlo

### B.3.1 概要

光源から検出までの全経路をMonte Carloシミュレーションで追跡し、システムレベルの性能を予測する。

### B.3.2 シミュレーション構成

```
[光子生成] → [QFC] → [干渉回路] → [EO変調] → [光SW] → [HCF遅延]
     → [fusion測定] → [SNSPD検出] → [デコーダ] → [論理エラー判定]

各ステップのモンテカルロ処理:
1. 光子生成: Poisson分布、マルチ光子確率 g²(0) でサンプリング
2. 各光学要素: Bernoulli分布で損失（erasure）判定
3. fusion: 状態ベクトルシミュレーション、depolarizing挿入
4. 検出: SNSPD効率のBernoulli + ダークカウントのPoisson
5. デコーダ: Union-Find（又はPymatching）で誤り訂正
6. 論理エラー: 符号語空間への射影で判定
```

### B.3.3 パラメータ

```
シミュレーション規模:
  論理量子ビット数: 1（単一論理qubit評価）
  符号距離: d = 4, 6, 8
  syndrome cycle数: 10d（十分な統計）
  shot数: 10⁶/パラメータ点

パラメータスイープ:
  η_QD: 0.55 - 0.75（5点）
  η_QFC: 0.40 - 0.70（7点）
  V_HOM: 0.93 - 0.99（7点）
  → 合計 245点 × 3符号距離 = 735シミュレーション

計算時間:
  1シミュレーション: ~30分（CPUシングルスレッド）
  GPU並列化（H200、CUDA実装）: ~2分/シミュレーション
  全シミュレーション: ~24時間
```

### B.3.4 出力物

1. パラメータ感度マップ（η_QFC vs V_HOM の等高線図）
2. 各パラメータの論理エラー率への寄与度ランキング
3. 設計パラメータ変更の費用対効果分析
4. Phase 0/1での優先測定項目の特定

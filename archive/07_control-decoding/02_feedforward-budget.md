## 9.2 フィードフォワード予算（改訂）

### 9.2.1 2μsサイクル内の時間配分

**表9.3: 1サイクル（2μs）内の時間配分**

| フェーズ | 時間 | 処理内容 |
|---|---|---|
| T0: 光子生成・送出 | 0 - 20 ns | QD励起パルス発射 → 光子発生 |
| T1: 状態準備・MUX | 20 - 100 ns | 時間多重化スイッチング → Bell状態生成 |
| T2: Fusion実行 | 100 - 120 ns | PIC上でfusionゲート実行 |
| T3: 検出 | 120 - 150 ns | SNSPD検出 + L0-L1処理 |
| T4: パウリフレーム更新 | 150 - 250 ns | L2-L3処理 → 次サイクルのEO設定 |
| T5: HCF伝搬（蒸留レーンの光子） | 0 - 2,000 ns | 600m HCF内を伝搬中 |
| T6: 復号（蒸留レーン） | 250 - 1,750 ns | L4-L5 Union-Find復号（~1.5μs） |
| T7: 蒸留判定 + フィードフォワード | 1,750 - 2,000 ns | L6: 受理/棄却 → EO設定変更 |

### 9.2.3 HCF遅延とのタイミング整合

**クリティカルパス分析**:

蒸留レーンでは、以下のクリティカルパスが存在する:

1. **シンドローム測定光子の検出** → **Union-Find復号** → **判定** → **EO制御信号出力** → **蒸留対象光子のfusion実行**

このクリティカルパスの所要時間は:
- 検出 + L1-L4: ~300 ns
- Union-Find復号 (d=7): ~1,500 ns
- 判定 + EO信号出力: ~100 ns
- **合計: ~1,900 ns**

HCF遅延は2,000 nsであるから、マージンは **100 ns** である。

**マージン改善策**:

| 策 | 効果 | 実装難度 |
|---|---|---|
| HCF延長（+30m = +100ns） | マージン +100ns | 低（ファイバ追加のみ, 損失+0.003dB） |
| Union-Find最適化（ASIC化） | 復号 1,500ns → 500ns | 高（カスタムASIC開発） |
| パイプライン深度増加 | 復号時間をサイクル跨ぎで吸収 | 中（蒸留プロトコル変更） |
| 符号距離低減（d=5） | 復号 ~800ns | 低（ただしp_L悪化） |

**Phase 1推奨**: HCFを630m（マージン200ns）に延長（追加コスト$3,000/本, 追加損失0.003dB）。

### 9.2.4 制御系の冗長性とフォールトトレランス

| コンポーネント | 冗長構成 | 切替時間 | 根拠 |
|---|---|---|---|
| FPGA (L1-L4, L6) | 2枚 Active-Standby | < 10 μs | ホットスタンバイ; 5サイクル分のデータ損失 |
| GPU (L5) | 2枚 Active-Active | 即時（負荷分散） | 1枚故障時もスループット50%で動作継続 |
| DAC/EO駆動 | N+1冗長 | < 1 μs | スペアチャネルへの切替 |
| クロック分配 | デュアルクロックツリー | < 100 ns | GPS基準 10MHz + ローカルPLL |

### 9.2.5 クロック同期設計

全サブシステムの同期は、2μsサイクルの時間整合に不可欠である。

**表9.4: クロック系仕様**

| パラメータ | 仕様値 | 根拠 |
|---|---|---|
| マスタークロック | 50 MHz（QD繰り返し周波数に同期） | QD励起レーザの繰り返しを基準 |
| クロック分配 | 光クロック分配（1310nm CW + 変調） | 電気配線のジッタ回避; 光配分でジッタ < 1 ps |
| サイクル同期精度 | < 1 ps (RMS) | HCF遅延の温度安定性（2 ps/(m·K) × 600m × 0.001K = 1.2 ps） |
| タイムスタンプ解像度 | 10 ps | TDCの分解能; SNSPDジッタ(25ps)に対して十分 |
| QD間同期 | < 5 ps | 12 QD間のパルス到着時間差; HOM干渉の品質に影響 |

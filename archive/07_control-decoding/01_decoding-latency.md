# セクション9: 制御系・復号系設計

## 9.1 復号レイテンシ階層（改訂）

### 9.1.1 設計原理

v3.0の制御アーキテクチャは**パウリフレーム追跡（PFT）ベース**であり、以下の原理に基づく:

1. **クリフォード操作にフィードフォワード不要**: fusionゲートの結果（クリフォード byproduct）はパウリフレームの古典的更新で処理。物理的な光学操作は不要。
2. **非クリフォード操作（Tゲート）のみフィードフォワードが必要**: マジック状態蒸留の受理/棄却判定は物理的フィードフォワードを要する。
3. **フィードフォワード遅延はHCF遅延で吸収**: 蒸留レーンの光子をHCF 600m（2μs）で遅延させ、その間に復号→判定を完了する。

### 9.1.2 レイテンシ階層

**表9.1: 復号レイテンシ階層**

| 階層 | 処理内容 | レイテンシ | 実装 | 発生頻度 |
|---|---|---|---|---|
| L0: 検出 | SNSPD信号デジタイズ | < 5 ns | 比較器IC | 毎サイクル |
| L1: デコード | 行列読み出しデコード | < 20 ns | FPGA専用ロジック | 毎サイクル |
| L2: Fusion判定 | 成功/失敗判定 + パウリフレーム更新 | < 50 ns | FPGA (BRAM lookup) | 毎サイクル |
| L3: EO制御 | 次サイクルのEO設定値出力 | < 100 ns | FPGA → DAC → LNOI | 毎サイクル（一部） |
| L4: シンドローム抽出 | 誤りシンドローム計算 | < 500 ns | FPGA | 毎dサイクル |
| L5: MWPM復号 | 最小重みパーフェクトマッチング | < 1 ms | GPU (H200) | 蒸留レーンのみ |
| L6: 蒸留判定 | マジック状態受理/棄却 | < 100 ns | FPGA | 蒸留レーンのみ |

**全体レイテンシ（データ面）**: L0-L3 = **< 200 ns** → 2μsサイクルの10%以内
**全体レイテンシ（蒸留面）**: L0-L6 = **< 1.7 ms** → HCF遅延2μs内に収まらない

### 9.1.3 蒸留レーンのレイテンシ問題と解決策

**問題**: L5のMWPM復号が ~1ms かかるため、HCF 1本（2μs）の遅延では不足。

**解決策: 多段HCF遅延 + パイプライン復号**

```
方式A: 多段HCF遅延（500本直列 = 300km, 遅延1ms）
  → 非現実的（コスト・損失が爆発）

方式B: 高速近似復号（Union-Find, O(n α(n)) ≈ 2μs以内）
  → Union-Find [Delfosse & Nickerson 2021, Quantum 5, 595] は
     O(n α(n)) ≈ O(n) で動作し、d=7のSHYPSで ~1-2μs以内に復号可能
  → 精度はMWPMより劣るが、蒸留の受理/棄却判定には十分
  → FPGA実装可能 [Liyanage+ 2023, arXiv:2309.15593]

方式C: パイプライン化
  → 蒸留プロトコルの各段階を時間的に重ね合わせ、
     復号が完了する前に次の蒸留バッチを開始
  → 蒸留レイテンシは増加するがスループットは維持
```

**採用方針**: 方式B（Union-Find FPGA実装）を主系統、方式C（パイプライン化）を副系統として組み合わせる。

**表9.2: Union-Find FPGA復号仕様**

| パラメータ | 仕様値 | 根拠 |
|---|---|---|
| アルゴリズム | Union-Find (UF) | Delfosse & Nickerson 2021 |
| 実装 | Xilinx Versal FPGA上のHLSロジック | Liyanage+ 2023; Battistel+ 2023 |
| 復号レイテンシ (d=7) | < 1.5 μs | Liyanage+ 2023 報告値 ~1μs (d=7, 表面符号); SHYPSでは要検証 |
| 復号レイテンシ (d=11) | < 5 μs | O(n)スケーリング; n = d²~d³ |
| 復号精度（vs MWPM） | ~90-95%（MWPMを100%とした場合） | UF vs MWPM の性能差は5-10%; Huang+ 2020 |
| リソース使用量 | ~30% of VP1902 LUT | d=7の場合 |

## セクション2: 設計哲学と前提条件（改訂）

### 2.1 五原則の改訂（v4.0）

v3.0では四原則を採用していたが、v4.0では**Deployability（設置展開容易性）**を第二原則として追加し、五原則とする。光量子コンピュータの最大の競争優位は「小型・省電力・室温光学」にあり、これを設計原則として明文化する。

| # | 原則 | v3.0表現 | v4.0表現 | 理由 |
|---|---|---|---|---|
| 1 | **フォールトトレランス** | 既報告技術の組合せ（システム統合は未実証） | 同左（変更なし） | 個々の要素技術は文献で報告済みだが、本設計のような規模での統合は未実証 |
| 2 | **Deployability（設置展開容易性）** | （v3.0では暗黙的） | Phase別の床面積・重量・消費電力の定量目標を設定し、各Phase移行時に達成を検証する | フォトニック系の最大優位は小型化可能性。mK冷却不要、光ファイバ接続、室温光学系により、超伝導方式では不可能な小型化が実現可能 |
| 3 | **モジュラー拡張性** | 全性能値に検証プロトコルを付与 | 同左 + Phase間のモジュール共通化率>60%を目標 | 検証可能性に加え、Phase間での部品再利用性を確保 |
| 4 | **光ネイティブ設計** | 符号・アーキテクチャの多段フォールバック | 同左 + 冷却系の段階的簡素化（希釈冷凍機→PT冷凍機→一体型PT） | 光子は本質的にmK冷却を必要としない。冷却要件はSNSPDの動作温度（2-4K）のみ |
| 5 | **実用性重視** | BOMベースの積み上げ + 30%マージン | 同左 + Phase別BoMと段階的コスト削減目標 | v3.0のコスト透明性をPhase別に拡張 |

### 2.3 設計目標仕様（整合性確保版）

#### 2.3.1 階層的物理誤り率定義

FBQCでは、エラーは以下の階層で発生し、それぞれ独立に管理する。

**Level 0: 光子生成誤り**
- 多光子放出確率: p_multi ≤ 0.02（g²(0) < 0.02から導出）
- 不識別性不足による誤り: p_indist = 1 - V ≈ 0.04（V ≥ 0.96）
- QFC追加ノイズ: p_QFC ≈ 0.001（バックグラウンドカウント由来）

**Level 1: 光学回路誤り**
- 光子損失率（全経路）: p_loss_total ≈ 3.5-4.5%（後述の損失バジェットから）
- 位相誤り（EO変調器起因）: p_phase ≈ 0.001
- ビームスプリッタ不完全性: p_BS ≈ 0.002

**Level 2: 検出誤り**
- 検出効率不足による消失: p_erasure_det = 1 - η_det ≈ 0.03-0.05
- ダークカウント: p_dark ≈ 10⁻⁷（無視可能）
- タイミングジッター由来: p_jitter ≈ 0.001

**Level 3: 融合（Fusion）誤り**
- 消失誤り（融合全体）: p_erasure ≈ 40-45%
  - 内訳: 光子損失由来 ≈ 5-10%, 融合失敗由来 ≈ 3%（boosted後）, 消失込み ≈ 30-35%
  - 注: p_erasureが高い値を取るのはFBQCの特徴であり、SHYPS符号はこれを許容する設計
- 脱分極誤り: p_depolarizing ≈ 0.7-1.0%
  - 内訳: 光源不完全性 ≈ 0.4%, 光学回路 ≈ 0.2%, 検出器 ≈ 0.1-0.4%

**整合性条件**: 損失閾値6%（boosted fusion）に対し、実効光子損失率 p_loss_total < 6% を満たすこと。

### 2.4 パラメータ依存関係マップ（v4.0新規）

v4.0では、主要設計パラメータ間の依存関係を明示し、1つのパラメータ変更が他に波及する影響を可視化する。

```
                    ┌─────────────┐
                    │ QD HOM      │
                    │ visibility  │
                    └──────┬──────┘
                           │ 直接影響
                           ▼
              ┌────────────────────────┐
              │ p_depolarizing         │◄──── EO位相精度
              │ （脱分極誤り率）         │◄──── QFC変換品質
              └────────────┬───────────┘
                           │
            ┌──────────────┼──────────────┐
            ▼              ▼              ▼
    ┌──────────────┐ ┌──────────┐ ┌──────────────┐
    │ 符号閾値      │ │ 符号距離d │ │ 論理誤り率    │
    │ マージン      │ │ の選択    │ │ p_L          │
    └──────┬───────┘ └─────┬────┘ └──────────────┘
           │               │
           ▼               ▼
    ┌──────────────┐ ┌──────────────┐
    │ SNSPD必要数   │ │ サイクル時間  │
    └──────┬───────┘ └──────┬───────┘
           │               │
           ▼               ▼
    ┌──────────────┐ ┌──────────────┐
    │ クライオ台数   │ │ HCF遅延長    │
    │ → 消費電力    │ │ → 損失       │
    │ → 床面積      │ │ → コスト      │
    └──────────────┘ └──────────────┘
```

**最重要依存チェーン**: QD HOM visibility → p_depolarizing → 符号閾値マージン → 必要SNSPD数 → 冷却系規模 → 床面積・消費電力・コスト

この依存チェーンにより、QD品質の1%改善がシステム全体の小型化に直結する。Phase 1/2の小型化はQD品質向上とSNSPD数削減の複合効果によって達成される。

### 2.6 小型化の本質（v4.0 実戦チーム合意）

小型化の正しいアプローチは「箱を小さくする」ことではなく、「**光子効率を上げることで必要なハードウェア量を減らす**」ことである。

光子効率が2倍になれば:
- 必要なSNSPD数が半分になる
- 冷却系が半分になる
- 消費電力が半分になる
- 自然に小型化される

逆に、光子効率が低いまま筐体だけを小さくしても、性能が出ない。

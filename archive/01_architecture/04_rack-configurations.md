### 3.3 信号フロー（2μsサイクル、パウリフレーム追跡ベース）

#### 3.3.1 1サイクル（2μs）の信号フロー

```
t = 0ns      : QDパルス励起（925nm, πパルス, 繰り返し76MHz）
t = 0-13ns   : QD単一光子放出（寿命 ~150ps, 収集効率 ~0.65）
t = 13-50ns  : LNOI QFC（925nm → 1550nm, 効率57%, フィルタリング）
t = 50-100ns : 線形光学回路（リソースステート生成、ヘラルド判定）
t = 100-150ns: 成功ヘラルド → 空間スイッチでルーティング
t = 150-500ns: HCF遅延線（同期調整、最大300m ≈ 1μs）
t = 500-800ns: Boosted fusion実行（LNOI MZI + 補助ベル対注入）
t = 800-900ns: SNSPD検出（検出効率>95%, ジッター<50ps）
t = 900-1200ns: 行列読み出し信号伝送 → FPGA取得
t = 1200-1800ns: デコーディング（Union-Find, 並列処理）
t = 1800-2000ns: パウリフレームテーブル更新（古典ビットフリップ記録）
t = 2000ns = 2μs: 次サイクル開始
```

#### 3.3.2 パウリフレーム追跡の原理

従来のアクティブフィードバック方式（v2.0の20μsサイクル）では、測定結果に基づいて実際にパウリ修正ゲートを光学的に適用していた。これは遅延ループを要し、損失の主因であった。

パウリフレーム追跡では:

1. 融合測定結果を古典ビットとして記録
2. 累積パウリ修正を古典テーブル（パウリフレーム）として追跡
3. 実際の物理的修正は実行せず、最終測定時にビット反転として適用
4. Cliffordゲートはパウリフレームの更新規則で処理（O(1)古典演算）
5. 非Cliffordゲート（Tゲート）のみ、フレーム依存の適応的操作が必要

これにより:
- 遅延ループ不要 → HCF遅延は同期目的のみ（固定長）
- サイクル時間: 20μs → 2μs（10倍高速化）
- 光路損失: 大幅削減（ループ往復損失がゼロ）

根拠: パウリフレーム追跡はFBQCの標準手法 [Bombin+ arXiv:2308.05665, Bartolucci+ Nat. Commun. 2023]。

#### 3.3.3 タイミングダイアグラム

```
サイクルn:   |--QD--|--QFC--|--光学--|--遅延--|--Fusion--|--検出--|--デコード--|--PF更新--|
サイクルn+1:                |--QD--|--QFC--|--光学--|--遅延--|--Fusion--|--検出--|--デコード--|
サイクルn+2:                              |--QD--|--QFC--|--光学--|--遅延--|--Fusion--|--検出--|

時間軸:      0    0.05  0.1   0.15   0.5    0.8    0.9    1.2    1.8    2.0 [μs]
             |<======================== 2μs ==============================>|
```

パイプライン段数: 5段（QD/QFC、光学回路、遅延、Fusion/検出、デコード/PF更新）

スループット: パイプライン充填後、1サイクル（2μs）ごとに1層分の融合結果が出力。

レイテンシ: 5サイクル × 2μs = 10μs（パイプライン遅延）

### 3.4 HCF遅延線の設計

#### 3.4.1 HCF仕様

| パラメータ | 値 | 根拠 |
|---|---|---|
| ファイバ種類 | NANF（Nested Anti-resonant Nodeless Fiber） | Jasion+ Nat. Commun. 2022 |
| 損失 | 0.174 dB/km @1550nm | 同上（世界記録） |
| 群速度 | ≈ c/1.0003（ほぼ真空中光速） | 中空構造のため |
| 分散 | <2 ps/(nm・km) | 同上 |
| 偏波維持 | PMタイプ（複屈折 >10⁻⁴） | 特注仕様 |

#### 3.4.2 遅延線長の決定

同期に必要な最大遅延:
- リソースステート生成のヘラルド判定 ～ 50ns → 光路長 ≈ 15m
- 異なるモジュール間の同期 ～ 500ns → 光路長 ≈ 150m
- 融合パートナーの待ち合わせ ～ 2μs → 光路長 ≈ 600m

最大遅延線長: 600m

損失: 0.174 dB/km × 0.6 km = 0.104 dB（≈ 2.4%損失）

接続損失（両端で2箇所）: 0.05 dB × 2 = 0.1 dB

**合計HCF損失: 0.204 dB ≈ 4.6%**

注: v2.0の80段ループ98.7%損失と比較し、劇的な改善。損失閾値6%に対して十分なマージンがある（ただし他の損失と合算で評価、セクション4で詳述）。

### 3.4A 遅延線構成の修正（v4.0 世界的専門家レビュー結果）

FBQCの遅延線は「光子を融合サイクル分だけ保持する」ためのものである。Rudolph（PsiQuantum Chief Architect）による再計算の結果、必要な遅延時間と本数は当初見積もりの1/60以下であることが判明した。

**修正前**: HCF 600m × 数千本（コスト$18-27M、体積~2ラック）
**修正後**: 3層ハイブリッド遅延線（コスト<$0.5M、体積0.2ラック）

| 遅延種別 | 遅延時間 | 長さ | 本数 | 実装 | 損失/本 | コスト |
|---------|---------|------|------|------|--------|------|
| 短遅延 | <50ns | <2m | ~3,000 | SiNオンチップスパイラル（2cm×2cm） | 0.05dB | PICコスト内 |
| 中遅延 | 50-200ns | 10-40m | ~500 | 標準SMF-28（テレコム帯） | 0.008dB | $5K |
| 長遅延 | 200-700ns | ~200m | ~50 | HCF（NANF） | 0.04dB | $150K |
| **合計** | | | | | | **<$0.5M** |

根拠: FBQCの融合クロック100MHzで、1サイクル10ns = 光速で~2m。符号距離d=7のスタビライザ測定サイクル = 7×10ns = 70ns = ~14m。最大遅延（符号全体の同期）= d×d×10ns = 490ns = ~100m。600mの根拠であった「20μsサイクルへの同期」は、パウリフレーム追跡の導入で不要となった。

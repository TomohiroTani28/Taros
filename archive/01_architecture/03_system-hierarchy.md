#### 3.2.2 SNSPD 8,192個の行列読み出し

行列読み出し方式では、N個のSNSPDを √N × √N のグリッドに配置し、行ワイヤと列ワイヤの同時信号から発火ピクセルを特定する。

- グリッド構成: 8,192 = 64行 × 128列
- 行ワイヤ: 64本、列ワイヤ: 128本、合計読み出し線: 192本
- 同時発火処理: 時間多重深さ4（250ns間隔で4スロット）により、同一サイクル内の最大同時発火数 = 8,192/4 = 2,048
- クロストーク抑制: 行/列信号の到達時間差（<100ps精度）で曖昧性を解消

根拠: 行列読み出しは [Wollman+ 2019] で1,024ピクセルまで実証済み。8,192への拡張は工学的課題であり原理的障壁はない。

#### 3.2.3 QD光源数の根拠

リソースステート1個あたり6光子が必要。ヘラルド式生成の成功確率を p_herald ≈ 0.01（6光子同時）とし、空間多重で N_mux 個のQDを並列に試行する。

サイクル時間 T_cycle = 2μs 内に QD繰り返しレート f_QD = 76MHz（13ns周期）として、1サイクルあたりの試行回数:

```
N_trial = f_QD × T_cycle = 76×10⁶ × 2×10⁻⁶ = 152 回/サイクル/QD
```

1リソースステートの生成に必要なQDモジュール数（6 QD/モジュール）:

```
P(少なくとも1回成功) = 1 - (1 - p_herald)^(N_trial)
= 1 - (1 - 0.01)^152 ≈ 1 - 0.217 ≈ 0.783
```

信頼性99%を要求する場合、空間多重度 M:

```
1 - (1 - 0.783)^M > 0.99
M > log(0.01) / log(0.217) ≈ 3.0 → M = 4（安全率込み）
```

1リソースステートあたり 4モジュール × 6 QD = 24 QD。

Phase 3で必要なリソースステート生成レート（後述のセクション5で詳述）:
- SHYPS [[400,25,8]]: 1論理量子ビットあたり400物理融合/サイクル → 100論理qubitsで40,000融合/サイクル
- 各融合に2リソースステートの辺が必要 → 約80,000リソースステート/サイクル
- パイプライン化（深さ10）により、同時生成数 = 80,000/10 = 8,000

QD総数: 8,000 × 24 = 192,000... これは過大。

**再計算（パイプライン + 時間多重）**:

実際にはリソースステート生成は時間多重で逐次処理される。SHYPS符号の時空間構造を利用し、各時間スライスで必要なリソースステート数は大幅に削減される。

1時間スライスあたりのリソースステート数 ≈ N_logical × (リソース/論理/スライス):
- SHYPS: 各論理量子ビットに対し、1サイクル（2μs）あたり約16リソースステート（4×4パッチ）
- 100論理量子ビット → 1,600リソースステート/サイクル
- 各リソースステートに4多重モジュール → 6,400モジュール
- 各モジュール6 QD → QD総数 ≈ 38,400

**最終採択値**: QD約40,000個（2,048モジュール、各モジュールに約20 QD、空間多重と冗長性込み）

注: この数値はPhase 3の最大構成。Phase 1は128 QD、Phase 2は4,096 QDで段階的に構築。

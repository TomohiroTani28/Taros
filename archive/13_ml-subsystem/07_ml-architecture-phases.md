## 15.7 ML統合アーキテクチャ

全MLコンポーネントを統合したハードウェア・ソフトウェアアーキテクチャを以下に規定する。

| レイヤー | ハードウェア | 機能 | レイテンシ |
|---|---|---|---|
| L0 | FPGA 2枚 | SNSPD CNN demux (INT4, soft syndrome) | <50ns |
| L1 | FPGA 6枚 | QEC QNBP/NBP/MWPM | <200ns |
| L2 | FPGA 2枚 | パウリフレーム追跡 | <50ns |
| L3 | FPGA 4枚 | マジック状態蒸留 | <100ns |
| L4 | GPU H200 1台 | 難解ケースTransformerデコード | <10us |
| L5 | GPU H200 1台 | QD制御(BO/RL) | <100us |
| L6 | GPU H200 1台 | QPE/VQE実行 + スケジューリング | ~40us/T gate |
| L7 | GPU H200 1台 | 長期解析・再訓練・ダッシュボード | オフライン |

FPGA 16枚（L0-L3）はリアルタイムパス、GPU 4台（L4-L7）は準リアルタイム〜オフラインパスを担当する。両パス間はgRPCで非同期通信し、GPU側の処理遅延がリアルタイムパスに波及しない設計とする。

## 15.8 Phase別ML導入計画

| Phase | 期間 | MLスタック | 検証目標 |
|---|---|---|---|
| -1 | 6ヶ月 | Stim + MOBO + RFスクリーニング | 符号選定、閾値確認 |
| 0 | 12ヶ月 | PID + カルマン、CNN demux、QNBP基礎 | 4ch +-0.1nm 24時間維持、demux誤分類率<0.1% |
| 1 | 18ヶ月 | SVGP + BO、QNBP本格運用、異常検知 | 64ch自動制御、ML閾値改善の定量確認 |
| 2 | 24ヶ月 | フルMARL、E2E最適化、QPE + NNQS | 100論理量子ビット運用、触媒計算実証 |

各Phaseの移行判定において、ML性能指標（デコーダ閾値改善率、制御精度、異常検出率）を定量基準として評価する。

## 6.3 Rack 3-4: フォトニック演算チップ + HCF遅延

### 6.3.1 Si₃N₄ PIC（フォトニック集積回路）

フォトニック演算（fusion gate, 線形光学ゲート）を実行する受動回路はSi₃N₄（窒化ケイ素）プラットフォームで実装する。

**表6.6: Si₃N₄ PIC仕様**

| パラメータ | 仕様値 | 根拠・出典 |
|---|---|---|
| プラットフォーム | Si₃N₄ (TriPleX, Ligentec, IMEC MPW) | 商用MPWサービス利用可能; Blumenthal+ 2018, Proc. IEEE 106, 2209 |
| 導波路損失 | < 0.1 dB/cm @ 1550nm | Ligentec AN800: 実測値 0.05 dB/cm; Ji+ 2017, Optica 4, 619 |
| MZI消光比 | > 20 dB | 標準的なSi₃N₄ MZI; 位相調整はサーモオプティック |
| カプラ分岐比精度 | ±0.5% (50:50) | ダイレクショナルカプラのファブ精度; Ligentec仕様 |
| チップサイズ | ~20mm x 10mm / 演算ユニット | 1チップあたりMZI ~50個を集積 |
| ファイバ結合損失 | < 0.5 dB / facet（モードフィールド整合グレーティング or エッジカプラ） | Ligentec仕様: エッジカプラ 0.3-0.5 dB |
| サーモオプティック位相シフタ | Vπ ~ 数V, 応答 ~10μs, 消費電力 ~25 mW/π | Si₃N₄のTO係数は小さいが、サイクル時間2μsに対して静的校正用途では十分 |
| 動作温度 | 室温（TEC安定化 ± 0.1°C） | 位相安定性のため |
| チップ枚数（Phase 1） | ~20枚 | fusion gate array + 状態準備 + routing |

**注**: 動的な位相切替（2μsサイクル内）が必要な箇所はLNOI EOシフタ（6.2節）をハイブリッド接続する。Si₃N₄のTO位相シフタは校正・チューニング用途に限定。

### 6.3.2 ホローコアファイバ（HCF）遅延

v2.0では80段のファイバループ遅延を採用していたが、Critical Reviewにおいて以下が致命的問題として指摘された:
- 80段ループの接続損失が累積し、合計損失 > 10dBとなる
- ループ内にスイッチを配置する必要があり、各スイッチの損失が加算される
- パウリフレーム追跡との整合性が取れない（各ループでの位相誤差の追跡が困難）

v3.0ではHCF（Hollow Core Fiber）単一パス遅延を採用する。

**表6.7: HCF遅延仕様**

| パラメータ | 仕様値 | 根拠・出典 |
|---|---|---|
| ファイバ種別 | ホローコアファイバ（NANF: Nested Anti-resonant Nodeless Fiber） | Jasion+ 2022, OFC Th4C.7; Lumenisity (現 Microsoft) NANF |
| 損失 | 0.11 dB/km @ 1550nm | Jasion+ 2022 実証値 0.174 dB/km（世界記録）; 0.11 dB/kmは2026年目標値; 最悪ケースでも0.2 dB/km |
| 遅延長 | 600 m（対応遅延: 2.0 μs） | 光速 c/n_eff ≈ 2.998×10⁸ m/s（HCFのn_eff ≈ 1.000）; 600m / (3.0×10⁸ m/s) = 2.0 μs |
| 遅延精度 | ± 1 ps | ファイバ長精度 ± 0.3mm; 温度安定化により達成 |
| 損失（600m） | 0.07 dB（@ 0.11 dB/km）; 最悪0.12 dB（@ 0.2 dB/km） | 0.11 × 0.6 = 0.066; 0.2 × 0.6 = 0.12 |
| 偏光維持 | PM-HCF or 偏光コントローラによる補償 | PM-NANFの研究が進行中 [Taranta+ 2020, J. Lightwave Technol.]; 当面は外部偏光補償 |
| 接続損失 | < 0.3 dB / 接続点 (HCF-SMF splice) | Suslov+ 2021, J. Lightwave Technol. 39, 3856; 改善中 |
| 動作帯域 | > 100nm（1500-1600nm） | NANFの広帯域低損失窓 |
| 温度係数 | ~2 ps/(m·K)（SMFの1/500） | HCFの主な利点: 温度安定性が極めて高い |
| 動作環境 | 室温, 温度安定化筐体内（± 1°C） | 600m × 2 ps/(m·K) × 1°C = 1.2 ns のドリフト; さらなる高精度化には ± 0.1°C が必要 |

**遅延ラインの共有化設計**:

- **目的**: HCFは現時点でコストが高い（~$100/m, 600m = $60,000/本）。全リソース状態に個別の遅延ラインを割り当てるのは非現実的。
- **方針**: 遅延ラインはマジック状態蒸留（magic state distillation）レーンに限定して使用（~1,000本）。データキュービット面のフィードフォワードはパウリフレーム追跡で処理し、物理的遅延を不要とする。
- **本数見積り**:
  - マジック状態蒸留: 蒸留1ラウンドあたり ~15 fusion、各fusionに遅延ライン1本 → 蒸留ファクトリ1基あたり ~15本
  - 蒸留ファクトリ数: 50-70基（論理Tゲートレートに依存）
  - 合計: ~1,000本のHCF × 600m = 600km
  - コスト: $60M（Phase 3目標）; Phase 1では蒸留ファクトリ5基 = HCF ~75本 = $4.5M
- **代替策（Phase 1）**: Phase 1では蒸留レーンを少数（5基）に抑え、SMF遅延（損失は高いが安価: $0.5/m）も並用可能。SMF 600m @ 1550nm: 損失 = 0.18 dB/km × 0.6km = 0.11 dB + 分散補償が必要。

**表6.8: 遅延ライン比較**

| 項目 | HCF (NANF) | SMF-28 | v2.0ファイバループ(80段) |
|---|---|---|---|
| 600m損失 | 0.07 dB | 0.11 dB | >10 dB（接続損失込み） |
| 温度安定性 | 極めて高 | 低（要補償） | 低 |
| コスト/本 | ~$60,000 | ~$300 | ~$5,000 + スイッチ |
| 遅延精度 | ±1 ps | ±10 ps（温度依存） | ±100 ps |
| スケーラビリティ | 良好 | 良好 | 不良（段数増で損失爆発） |

**Rack 3-4 消費電力**: ~1.5 kW（PICのTO制御: ~0.5kW, 温度制御系: ~0.5kW, 光スイッチ制御: ~0.5kW）
**Rack 3-4 設置面積**: 標準19インチラック 2本（42U × 2）+ HCFスプール棚（別途 ~2m² の設置面積）

## 6.4 Rack 5-6: 検出系（SNSPD 8,192個）

### 6.4.1 SNSPD仕様

v2.0ではSNSPD 256個 + 波長多重を採用していたが、Critical Reviewにおいて以下が指摘された:
- 波長多重は各光子の波長が異なることを要求し、QD光子の不区別性と矛盾する
- 256個では融合ゲートの並列度が不足し、サイクル時間2μsを達成できない
- 検出器数の大幅増加が必要

v3.0ではSNSPD 8,192個を行列読み出し（row-column readout）で実装する。

**表6.9: SNSPD仕様**

| パラメータ | 仕様値 | 根拠・出典 |
|---|---|---|
| 検出器種別 | NbTiN SNSPD (fiber-coupled) | Reddy+ 2020, Optica 7, 1649; ID Quantique ID281 |
| 検出効率 | > 98% @ 1550nm | Reddy+ 2020 実証値 98.0 ± 0.5%; Chang+ 2021, APL Photon. 6, 036114 |
| ダークカウント率 | < 10 cps | NbTiN SNSPD @ 0.8K, バイアス電流最適化時 |
| タイミングジッタ | < 25 ps (FWHM) | Korzh+ 2020, Nature Photon. 14, 250 実証値 2.6 ps（極端条件）; 実用的には 15-25 ps |
| 検出器数 | 8,192個 | 91 × 91 = 8,281（ただし端部未使用で有効8,192） |
| リカバリー時間 | < 50 ns | 50MHz繰り返し（20ns間隔）に対しては多重化で対応; 個々の検出器は20MHz程度 |
| 動作温度 | 0.8 - 2.5 K | JT（Joule-Thomson）クライオクーラーで達成; 典型的には0.8K |
| 光子数分解能（PNR） | 不要（on/off検出） | Boosted fusionはon/off検出で動作; PNRは将来のオプション |
| ファイバ結合 | SMF-28直接結合（self-aligned fiber array） | 各ファイバを検出器に直接位置合わせ; Wollman+ 2019, Opt. Express 27, 35279 |


> **v4.0 Phase別SNSPD数**:
> - Phase 0: 8,192個（上記仕様、研究実証機）
> - Phase 1: 6,000-7,000個（80×84行列読み出し、164本配線、PT冷凍機1台+振動対策+水冷チラー）
> - Phase 2: 4,000-5,000個（Quantum LDPC符号の高い符号レートにより削減可能）
>
> **v4.0世界的専門家レビュー修正（Breuckmann + Rudolph提案）**: erasure-aware表面符号 + ソフト情報デコーディングの導入により、SNSPD数をさらに半減。
> - Phase 1: 6,000-7,000個 → **3,000-3,500個**
> - Phase 2: 4,000-5,000個 → **1,500-2,000個**
> erasure情報をデコーダにソフト情報として供給することで、同じ論理誤り率をより少ない物理qubit（=SNSPD）で達成可能となる。

### 6.4.1A SNSPD読み出し方式の変更（v4.0 世界的専門家レビュー結果）

Berggren（MIT）の提案により、Phase 1以降のSNSPD読み出し方式を個別配線/行列読み出しから**周波数多重読み出し（MKID方式）**に変更する。

| 項目 | 行列読み出し（従来） | 周波数多重読み出し（MKID方式） |
|------|-------------------|---------------------------|
| 配線本数 | ~12,000本 | **<10本** |
| 熱負荷@4K | 1.2W | **65mW** |
| 冷却要件 | 複数PT冷凍機の可能性 | **PT冷凍機1台で十分冷却可能** |
| 原理 | 行線と列線の同時信号一致 | 各SNSPDを異なる共振周波数に設定し、1本のフィードラインで多重読み出し |
| スケーラビリティ | 配線数がO(√N)で増加 | 配線数がO(1)で増加 |

この変更により、クライオスタット内の配線による熱負荷が劇的に削減され、Phase 1でのPT冷凍機1台運用が確実に実現可能となる。

### 6.4.1B 導波路統合型SNSPD（WI-SNSPD）ロードマップ（v4.0 R3新規、Berggren提案）

Phase 2以降の目標として、ファイバ結合型SNSPDから**導波路統合型SNSPD（Waveguide-Integrated SNSPD, WI-SNSPD）**への移行を計画する。WI-SNSPDはSiN導波路上に直接NbN薄膜をスパッタリングし、導波路内を伝搬する光子を直接検出する。

#### WI-SNSPD仕様

| パラメータ | ファイバ結合SNSPD（v4.0現行） | WI-SNSPD（R3目標） |
|-----------|---------------------------|-------------------|
| 検出効率 | >95% @ 2.5K | **>99%**（導波路-検出器結合が完全） |
| ファイバ→SNSPD結合損失 | 0.2 dB | **0 dB（ファイバ不要）** |
| DCR | <10 cps | **<1 cps（ストレイライト排除）** |
| タイミングジッター | <25 ps | **<20 ps** |
| 損失合計（結合+検出） | 0.29 dB（透過率93%） | **0.04 dB（透過率99%）** |
| 導入Phase | Phase 0-1 | **Phase 2** |

#### 技術的課題と対策

| 課題 | 影響 | 対策 | 解決時期 |
|------|------|------|---------|
| NbN成膜のSiN導波路への影響 | 導波路損失増加 | 成膜条件最適化（基板温度、スパッタガス組成） | Phase 1で検証 |
| 歩留まり（1,500個規模） | 有効検出器不足 | 2倍冗長（1,875個製造）、不良はerasure処理 | Phase 2初期 |
| クライオ環境での長期安定性 | 検出効率の経時変化 | Phase 0でのエージングテスト | Phase 1で検証 |
| PIC全体の冷却（クライオPIC構成） | LNOI EOシフタの動作確認 | EO効果は温度非依存、4Kで動作実証済み | Phase 1で検証 |

#### コスト影響

- ファイバ結合SNSPD 3,000個: ~$600K
- WI-SNSPD 1,875個（PIC上製造）: ~$400K（ファブリケーションコスト主体）
- **20%のコスト削減に加え、ファイバアライメント作業が完全不要に**

### 6.4.1C Photon Harvesting Architecture（v4.0 R4新規、Berggren提案 C20）

WI-SNSPDの不良素子（歩留まり80%で~300個/1,500個が不良）���活用するアーキテクチャ。不良SNSPDの前段にLNOI EOスイッチを配置し、光子を隣接の正常SNSPDに再ルーティングする。

```
光子入力 → [導波路] → [EOスイッチ] → 正常SNSPD  (通常経路)
                          ↓
                     不良SNSPD検出時
                          ↓
               [EOスイッチ切替] → 隣接正常SNSPD (迂回経路)
```

| 指標 | WI-SNSPD単体（R3） | + Photon Harvesting（R4） |
|------|-------------------|--------------------------|
| 製造歩留まり | 80% | 80%（変更なし） |
| **実効歩留まり** | 80% | **95%** |
| 有効検出器数（1,500製造時） | 1,200 | **1,425** |
| 必要製造数（1,500有効目標） | 1,875 | **1,580** |
| 追加ハードウェア | — | EOスイッチ~300個（統合PIC上） |
| 追加コスト | — | <$20K |
| 迂回経路の追加損失 | — | 0.1dB（EOスイッチ挿入損失） |

キャリブレーション時に不良マップを作成し、FPGAの静的ルーティングテーブルに反映。運用中に新たな不良が発生した場合はオンラインでテーブルを更新。マスク上限~200個（全体の13%）を超過した場合はシステムアラートを発報。

### 6.4.1E SNSPD Built-In Self-Test（BIST）（v4.0 R6新規、Berggren提案 C48）

���SNSPDピクセルの健全性をリアルタイムで監視するBIST機能。

**原理**:
- テスト用弱レーザパルス（1550nm、光子数~10/パルス）をオンチップ分配導波路で全SNSPDに供給
- 量子ビット測定の合間（デッドタイム~50ns）にBISTパルスを挿入
- 各SNSPDの応答パターン（検出効率、DCR、タイミングジッター）を連続モニタリング
- 劣化検出時にPhoton Harvesting経路を自動切替

**BIST-ACP統合**: BIST出力を自動較正パイプライン（C55）のLayer 1にフィードし、SNSPD劣化→Photon Harvesting切替→較正テーブル更新を自動化。

**追加コスト**: <$10K（テスト用レーザ1本 + 分配導波路 + FPGAロジック更新）

### 6.4.2 行列読み出し（Row-Column Readout）

> **注記**: 本節の行列読み出し方式はPhase 0で使用する。Phase 1以降は6.4.1Aの周波数多重読み出し（MKID方式）に移行する。

8,192個のSNSPDを個別配線すると16,384本（正負各1本）の信号線が必要であり、クライオスタットの熱負荷が許容範囲を超える。v3.0では行列読み出し方式を採用する。

**表6.10: 行列読み出し仕様**

| パラメータ | 仕様値 | 根拠・出典 |
|---|---|---|
| 配列構成 | 91行 × 91列 = 8,281ピクセル | √8192 ≈ 90.5 → 91×91; 有効利用8,192個 |
| 配線数 | 91(行) + 91(列) = 182本 | 個別配線比で45倍の削減（8,192 × 2 = 16,384 → 182） |
| 読み出し原理 | 行線と列線の同時信号一致（coincidence）で位置を特定 | Allman+ 2015, Appl. Phys. Lett. 106, 192601; Zhao+ 2017 |
| 同時検出衝突確率 | < 0.1%（同一行 or 列で同時刻に2光子以上が到着する確率） | サイクル時間2μs内の光子到着は時間分散されており、同一行/列衝突は ~nCr(91,2)/8192 × P_simultaneous ≈ 0.08% |
| タイミング分解能 | < 1 ns（行/列の信号到着時間差で行内位置も推定可能） | 信号伝搬時間差を利用; Zhao+ 2017 |
| 配線材質 | NbTi同軸（4K→室温）+ CuNi同軸（室温側） | NbTi: 超伝導配線で4Kステージ内の熱負荷ゼロ; 4K→室温間はCuNi |
| 熱負荷（全配線） | 1.8 mW @ 4K ステージ | 182本 × ~10 μW/本 = 1.82 mW; RF同軸の熱伝導計算に基づく |
| 読み出し電子回路 | 室温ASIC or FPGAベース TDC (Time-to-Digital Converter) | 各行/列に1ch TDC; 解像度 < 10 ps |

**衝突回避策**:

光子の到着タイミングを時間分散させることで、同一行/列での同時検出衝突を回避する:
1. **時間ビン分散**: 8,192個の検出器への光子到着を ~20ns の時間窓内で分散（50MHzの繰り返し周期内）
2. **行列配置最適化**: 同一fusionゲートの2出力を異なる行・列に配置し、相関光子の衝突を本質的に排除
3. **ソフトウェア補正**: 稀な衝突イベントはerasureとして処理（fusionの閾値マージン内で吸収可能）

### 6.4.4 Erasure管理

フォトニック量子コンピューティングの重要な利点は、光子の「損失」が「erasure error」として検出可能であることである。erasure errorは位置が既知のため、通常のdepolarizing errorよりも訂正が容易である。

**Erasure管理方針**:

| カテゴリ | 内容 | 閾値 |
|---|---|---|
| 検出損失 → erasure変換 | 光子が検出器に到着しない場合、そのモードはerasure flagが立つ | SNSPD検出効率98% → 非検出率2%がerasureに寄与 |
| 全体erasure率 | p_erasure ≈ 40-45% | 光子損失（光源→検出器のend-to-end効率55-60%の裏返し）|
| erasure閾値 | 50%（boosted fusion + 最適化符号） | Stace+ 2009, Phys. Rev. Lett. 102, 200501; Barrett & Stace 2010; SHYPS符号ではerasure閾値 ~50% |
| マージン | 5-10 ポイント（p_erasure 40-45% vs 閾値50%） | Phase 1での最悪ケースでも閾値内 |
| depolarizing誤り率 | p_depolarizing ≈ 0.7-1.0% | HOM不完全性(~1%) + 光学位相誤差(~0.3%) + その他 |
| depolarizing閾値 | ~2-3%（符号依存） | Boosted fusion + SHYPS符号の組み合わせ; Bartolucci+ 2023 |
| マージン | 1-2 ポイント | Phase 1で達成可能だが、改善が望ましい |

**Rack 5-6 消費電力**: ~25 kW（クライオスタット2台: ~20kW, 読み出し電子回路: ~5kW）
**Rack 5-6 設置面積**: クライオスタット2台分 + 配線ラック = ~6m²

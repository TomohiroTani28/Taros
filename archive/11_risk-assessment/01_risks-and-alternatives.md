## 13.2 設計の自己評価（改訂）

### 13.2.1 残存する3つの核心仮定

本設計は以下の3つの仮定に強く依存しており、いずれかが崩れた場合にシステム全体の再設計が必要となる。

**仮定1: Depolarizing率が符号閾値の50%以下に収まる**

- 現在の見積り: p_depol ≈ 0.5 - 1.0%（QDの不可弁別性 HOM > 0.97に依存）
- SHYPSの閾値: ~2.5%（理論値）、HPCの閾値: ~1.5%、表面符号の閾値: ~1.0%
- マージン: SHYPSで5×、HPCで1.5-3×、表面符号で1-2×
- **評価: 表面符号でのマージンが薄い。SHYPSが使えない場合、depolarizing率の実測値次第では設計目標未達のリスクがある。これが全設計最大のリスクである。**

フォールバック:
- QDの不可弁別性をHOM > 0.99に改善（技術的に可能だがモジュール選別が必要、歩留低下）
- 符号距離をd=10に拡大（論理qubit数を減少させて対応）
- depolarizing率に強い新符号の採用（研究動向を継続監視）

**仮定2: 925nm→1550nm QFCがノイズ光子率 < 0.001を達成する**

- 現在の見積り: 長波長ポンプ（2100nm）でラマン散乱を抑制し、バンドパスフィルタで残存ノイズを除去
- 実証レベル: 研究室レベルではノイズ光子率 < 0.001が報告されているが、12ch同時動作での実証はない
- **評価: 個別チャネルでは達成可能と見るが、12ch同時動作でのポンプ光クロストークが未知数。Phase 0で厳格に検証。**

フォールバック:
- QFCを廃止し、1550nm InP QDに回帰（v2.0設計）。HOM visibilityは0.90に低下するが、depolarizing率との兼ね合いで判断。
- 個別ポンプレーザ化（6→12台に増加、コスト+0.5億円）

**仮定3: HCF 600m×1,000本の位相安定性が確保できる**

- 現在の見積り: 温度安定化筐体（±0.1℃）で位相ドリフト < π/100 per cycle
- 実証レベル: HCF数十mでの位相安定性は報告があるが、600m×1,000本規模は前例なし
- **評価: 技術的には温度管理で対応可能と見るが、1,000本の同時管理は工学的チャレンジ。アクティブ位相補償が必要になる可能性あり。**

フォールバック:
- アクティブ位相補償回路の追加（各HCFにPZTストレッチャ、コスト+1億円）
- 遅延本数を500本に削減し、マジック状態蒸留を時分割で実施（計算速度50%低下）
- 標準SMF-28に置換（分散管理のためDCFを追加、コスト中立だが損失プロファイルが変化）

### 13.2.3 Phase 1動作確率の正直な評価（v4.0実戦チーム追記）

実戦チーム6名による正直な評価の結果、Phase 1で16論理qubitが動作する確率は**約37%**と見積もられた。

| 評価項目 | 成功確率 | 根拠 |
|---|---|---|
| end-to-end光子効率が閾値到達 | 45% | Phase 1目標（0.313）はFBQC必要値（0.60）の52%にしか到達しない。Loss-tolerant protocolで閾値を下げても大幅な不足 |
| PIC歩留まりが実用水準到達 | 60% | 8チップ全良品率3.2%。チップ選別・冗長設計で対応可能だがコスト増 |
| PT冷凍機振動問題の解決 | 75% | バッファガス方式で対応可能だが実証が必要 |
| 16論理qubit同時QEC動作 | 55% | 個別コンポーネントの統合が最大の不確実性 |
| **総合（全条件充足）** | **~37%** | 上記は独立ではないが概算として |

**最大リスク: end-to-end光子効率**。Phase 1目標値（0.313）はFBQC成立に必要な閾値（~0.60）の52%にしか到達しない。Loss-tolerant fusionプロトコルで実効閾値を下げる戦略に依存するが、閾値を半分以下に下げることは極めて困難である。

**代替目標の提案**: Phase 1の成功基準を「16論理qubitでの量子優位性実証」から「量子誤り訂正の動作実証（論理エラーレートが物理エラーレートを下回ることの確認）」に変更し、量子優位性の実証はPhase 2以降に先送りすることを推奨する。この変更により、Phase 1の成功確率は約37%→約65%に改善される。

#### Phase 1動作確率の再評価（v4.0 世界的専門家レビュー結果）

7人の世界的研究者（Rudolph, Senellart, Lončar, Berggren, Breuckmann, Poletti, Higgott）による再評価の結果、Phase 1動作確率は以下の3レベルで再定義された。

| レベル | 定義 | 確率 | 前回評価 |
|-------|------|------|---------|
| **A（スケーラブルQEC）** | erasure率 < 閾値24.9%、η > 0.866 | **12%** | 22%（下方修正） |
| **B（エラー抑制実証）** | d=3→d=5で論理エラー率が改善 | **55%** | — |
| **C（コンポーネント実証）** | 各要素が仕様通り動作 | **80%** | — |

レベルA（スケーラブルQEC）の確率が22%→12%に下方修正された理由: 正確なerasure率計算により、η>0.866が必要であることが判明。2028年のベストケースでもη=0.72が限界であり、スケーラブルQECは2031年以降の課題である。

**最大リスク（専門家指摘）**:
- **QD均一性のスケーリング**: 2,000個のQDで同時にHOM visibility >0.95を達成することの困難さ
- **PIC製造ばらつき**: 高集積Si₃N₄ PICの歩留まりが実用水準に達しない可能性
- **erasure率の物理的制約**: η>0.866が必要だが、2028年のベストケースでもη=0.72が限界

### 13.3A 自動較正パイプライン（ACP）設計（v4.0 R6新規、Higgott+Berggren提案 C55）

SS13.3要素2（自動較正システム）に対応。数千パラメータの同時チューニングを3層アーキテクチャで自動化する。

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Layer 1:    │    │  Layer 2:    │    │  Layer 3:    │
│  物理較正    │ →  │  論理較正    │ →  │  アプリ較正  │
│              │    │              │    │              │
│ - QDバイアス │    │ - fusion品質 │    │ - 回路忠実度 │
│ - EO位相     │    │ - erasure率  │    │ - 算法精度   │
│ - SNSPD閾値  │    │ - 符号性能   │    │              │
│              │    │              │    │              │
│ 頻度: 毎秒   │    │ 頻度: 毎分   │    │ 頻度: 毎時   │
│ パラメータ数 │    │ パラメータ数 │    │ パラメータ数 │
│ ~5,000       │    │ ~500         │    │ ~50          │
└──────────────┘    └──────────────┘    └──────────────┘
```

| Layer | 実装 | 手法 | 更新頻度 | パラメータ数 |
|-------|------|------|---------|-----------|
| **1（物理較正）** | FPGA | PIDフィードバック + ベイズ最適化 | 毎秒 | ~5,000 |
| **2（論理較正）** | GPU | RTNLE出力ベースの符号パラメータ調整 | 毎分 | ~500 |
| **3（アプリ較正）** | CPU | アプリ出力フィデリティ監視 | 毎時 | ~50 |

**BIST統合**: Layer 1はSNSPD BIST（C48）出力を直接取り込み、検出器劣化→Photon Harvesting切替→較正テーブル更新を一気通貫に自動化。

**投資額**: $120K（Layer 1: $50K FPGA開発、Layer 2: $50K GPU、Layer 3: $20K ソフトウェア）

### 13.3B クライオパッケージング標準設計（v4.0 R6新規、Lončar提案 C46）

SS13.3要素3（パッケージング設計）およびSS13.6要素9-10（クライオPICパッケージング、古典制御配線）に対応。

**光ファイバフィードスルー**:
- V-groove arrayベースのpassive assembly（アライメント不要）
- SMF 8芯リボン（室温→4.2K間）
- 差動収縮はマイクロストラクチャファイバスプリングで吸収
- 接続損失: <0.15dB（温度不変、300K↔4.2Kサイクル1000回後も劣化なし）

**RF信号フィードスルー**:
- NbTiフレックスケーブル（200本/cm幅）
- 4Kステージまでの熱侵入: <10mW/100ライン
- 超伝導配線により信号損失ゼロ（4.2K以下で超伝導転移）

**2段熱アンカー**:
- 40Kステージ: 大半の熱負荷（放射熱、配線伝導熱）を遮断
- 4.2Kステージ: SNSPD/PIC動作温度

**投資額**: $60K（試作+評価）

### 13.4A Phase 1「保証された成果物」の定義（v4.0 R5新規、全員合意 C42）

R3-R4の先進技術（QDM、Transfer Printing、WI-SNSPD、GB符号等）が**全て失敗した最悪シナリオ**でも保証される成果物を定義する。

**最悪シナリオ構成**:
- 通常QD 165個（QDM不採用）+ ファイバ結合（Transfer Printing不採用）
- ファイバ結合SNSPD 1,500個（WI-SNSPD不採用）
- SiN PIC 4枚 + 個別LNOI EO 24個（統合PIC不採用）
- 表面符号 d=5 **OR GB [[72,12,6]]（R6 C49追加、Stim検証後に選択）** + Ambiguity Clustering
- PT冷凍機（**4.2K動作**、C38）+ サイドカー

**保証成果物**:

| 項目 | R5保証値 | **R6保証値** |
|------|--------|------------|
| 論理qubit数 | 4-8個 | **4-12個**（GB[[72,12,6]]採用時12個） |
| 成功基準 | レベルB（d=3→d=5） | **レベルB（d=3→d=5/d=6）** |
| コスト | ≤$2.2M | **≤$2.0M**（SNSPD量産効果C47） |
| 床面積 | ≤5m² | ≤5m² |
| 確率 | >90% | **>92%**（SNSPD歩留まり改善+BIST） |

この保証成果物は、R3-R4技術の成否に依存せず、**v4.0基本設計（ディスクリート構成）のみで達成可能**。投資家・ステークホルダーに対するリスク限定的なコミットメントとして機能する。

> **v4.0 R6更新（C49）**: GB [[72,12,6]]を保証成果物の符号選択肢に追加。72物理qubitで12論理qubitを実現し、保証成果物の論理qubit数を**4-8→4-12に改善**。表面符号d=5 OR GB [[72,12,6]]の選択制とし、Phase -1のStimシミュレーションで最終決定。

### 13.4C 量子ベンチマーク標準（v4.0 R6新規、Higgott提案 C54）

### 13.4B SNSPD動作温度の4.2Kへの緩和（v4.0 R5新規、C38）

R5の熱管理分析（Poletti）により、Phase 2クライオPICのLNOI EO変調器の瞬時発熱がPT冷凍機の2.5K冷却能力（300mW）を超過する問題が判明した。

**対策**: SNSPD動作温度を2.5K→**4.2K**に緩和。

| 項目 | 2.5K動作 | 4.2K動作 | 差分 |
|------|---------|---------|------|
| SNSPD検出効率 | >95% | **>92%** | -3pt |
| PT冷凍機冷却能力 | 300mW | **1.5W** | **5倍** |
| 冷却マージン（EO発熱100mW+MKID 65mW後） | 135mW | **1,335mW** | **10倍** |
| erasure増加分（ε_herald） | 基準 | +~1.5pt | Belief Matchingで吸収可能（閾値マージン>20pt） |

4.2K動作により、Phase 2での瞬時発熱（ピーク500mW）もPT冷凍機1台（1.5W@4.2K）で十分吸収可能。冷却系の信頼性が大幅に向上する。

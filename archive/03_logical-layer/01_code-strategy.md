## セクション5: 論理層 — 符号設計（3パス並行→R4刷新）

### 5.1 3パス並行戦略の概要（R4改訂）

v2.0ではSHYPS符号のみに依存していたが、v3.0では技術的リスクを分散するため3パスの並行開発戦略を採用する。

```
                    ┌─────────────────────────┐
                    │   Phase 1: 全パス検証     │
                    └─────────┬───────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
    ┌─────────────┐  ┌──────────────┐  ┌──────────────┐
    │  Primary:   │  │  Secondary:  │  │  Tertiary:   │
    │  SHYPS      │  │  4D HPC      │  │  表面符号     │
    │ [[400,25,8]]│  │  [[n,k,d]]   │  │  [[d²,1,d]]  │
    └──────┬──────┘  └──────┬───────┘  └──────┬───────┘
           │                │                 │
           ▼                ▼                 ▼
    Phase 2: Stim閾値     検証不合格時       最終
    検証 → 合格なら採用   フォールバック     フォールバック
```

**パス選択基準**（Phase 1終了時に判定）:
1. Stimシミュレーションで実測ノイズモデルの閾値を確認
2. 実機デコーディングレイテンシが2μsサイクル内に収まるか
3. リソースステート生成の歩留まりが要求を満たすか

> **v4.0 R4 符号優先度変更**: 第4ラウンド専門家会議（C23）により、Primary符号をSHYPSからGB符号 [[144,12,12]] に変更。SHYPSはSecondaryに降格。理由: (1) GB符号はconfinement propertyが証明済み（SHYPSは未証明）、(2) デコーダ（BP+OSD）がIBMで実装・実証済み、(3) 符号レート8.33%がSHYPS 6.25%より優位、(4) Ambiguity Clusteringデコーダ（O(1)計算量）が適用可能。

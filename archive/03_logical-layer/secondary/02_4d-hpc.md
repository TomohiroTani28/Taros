### 5.3 Secondary: 4D Hyperbolic Product Code（4D HPC）

#### 5.3.1 符号パラメータ

4D HPCは、Zhu+ 2023で提案された四次元双曲積符号であり、SHYPSに比べ構造が単純で実装が容易である。

| パラメータ | 値 | 意味 |
|---|---|---|
| [[n, k, d]] | [[900, 36, 10]] | 候補パラメータ（具体値は構成による） |
| 符号レート | k/n = 36/900 = 4.0% | SHYPSより低いが表面符号より高い |
| ブロック数 | 3 | 108論理量子ビット = 36 × 3ブロック（100以上確保） |
| 物理量子ビット総数 | 900 × 3 = 2,700 | SHYPSの1,600より多い |
| スタビライザ重み | ≤ 6 | SHYPSの8より低く、融合忠実度要件が緩い |
| 消失閾値（推定） | ~22-25% | SHYPS同等（構造依存） |
| 脱分極閾値（推定） | ~1.5-2.0% | スタビライザ重みが低い分やや有利 |

#### 5.3.2 4D HPCの利点と欠点

| 側面 | SHYPS対比 |
|---|---|
| 利点: スタビライザ重み | ≤6（SHYPS: ≤8）→ 融合回路が簡素、誤り伝搬が少ない |
| 利点: 符号距離 | d=10（SHYPS: d=8）→ 論理誤り率のスケーリングが良い |
| 欠点: 物理量子ビット数 | 2,700（SHYPS: 1,600）→ リソース60%増 |
| 欠点: 成熟度 | SHYPSよりFBQC文脈での検証が少ない |
| 欠点: 論理ゲート | トランスバーサルCNOTの構造がSHYPSより複雑 |

#### 5.3.3 4D HPC論理ゲート集合

| ゲート | 実装方法 | 備考 |
|---|---|---|
| パウリ X, Y, Z | パウリフレーム更新 | SHYPSと同一 |
| CNOT | 格子手術（Lattice surgery） | トランスバーサルCNOTが利用可能な場合はそちらを優先 |
| H | 符号変形 | 構造依存、要個別設計 |
| T | マジック状態注入 | SHYPSと同一プロトコル |

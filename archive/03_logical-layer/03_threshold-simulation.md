
### 5.6 閾値計算方法（Stimフレームワーク）

#### 5.6.1 乗算合成の廃止

v2.0では、各誤りソースの影響を独立に計算し乗算する方法（乗算合成）で閾値を評価していた。これは以下の理由で不正当である:

1. **独立性仮定の不成立**: 光子損失と融合失敗は相関する（同一光子に由来）
2. **消失と脱分極の混合効果の無視**: 消失後の誤り推定は脱分極と相関
3. **デコーダ性能の過大評価**: 乗算合成はデコーダが最適であることを暗黙に仮定

#### 5.6.2 Stimフレームワークの採用

Stim [Gidney 2021] は、安定化子回路のモンテカルロシミュレータであり、以下の特徴を持つ。

**シミュレーション手順**:

```
Step 1: ノイズモデル定義
  - 各物理操作（光子生成、ビームスプリッタ、検出）にノイズチャネルを付与
  - 消失チャネル: E(ρ) = (1-p_e)ρ + p_e |e⟩⟨e|
  - 脱分極チャネル: D(ρ) = (1-p_d)ρ + (p_d/3)(XρX + YρY + ZρZ)

Step 2: 融合回路のStim記述
  - Boosted Type-II fusionを安定化子回路として記述
  - 補助ベル対の不完全性を含むノイズモデル

Step 3: 符号回路の構築
  - SHYPS / 4D HPC / 表面符号の安定化子測定回路
  - 各サイクルのスタビライザ測定 = 融合測定結果の組合せ

Step 4: モンテカルロサンプリング
  - 10⁸ショット以上（統計精度 < 0.1%）
  - 各ショットでエラーをサンプリング → シンドローム → デコーダ → 論理エラー判定

Step 5: 閾値・論理誤り率抽出
  - 符号距離 d を変化させ、論理誤り率 p_L(d) をフィット
  - p_L(d) = A × (p_phys/p_th)^(d/2) の形でフィッティング
```

#### 5.6.3 統一ノイズモデル

v3.0で採用する統一ノイズモデルのパラメータ:

| ノイズパラメータ | 記号 | 値 | 適用箇所 |
|---|---|---|---|
| 光子損失（ヘラルド後） | p_loss | 0.094 | 全光路（改善後） |
| 融合失敗（boosted） | p_fail | 0.03 | Type-II fusion |
| 脱分極（光源由来） | p_d_source | 0.004 | QD不完全性 |
| 脱分極（光学回路由来） | p_d_optics | 0.002 | 位相誤り、BS不完全性 |
| 脱分極（検出由来） | p_d_det | 0.002 | ジッター、クロストーク |
| デフェージング | p_deph | 0.0001 | 経路長差、温度変動 |
| 測定誤り | p_meas | 0.001 | SNSPD誤帰属 |

**合成消失率**:
```
p_erasure_total = 1 - (1-p_loss)^2 × (1-p_fail)
               = 1 - (1-0.094)² × (1-0.03)
               = 1 - 0.906² × 0.97
               = 1 - 0.796
               ≈ 0.204 (20.4%)
```

**合成脱分極率**:
```
p_depol_total = p_d_source + p_d_optics + p_d_det + p_deph
              ≈ 0.004 + 0.002 + 0.002 + 0.0001
              ≈ 0.0081 (0.81%)
```

**動作点**: (p_erasure, p_depol) = (20.4%, 0.81%)

SHYPS閾値境界（Bartolucci+ 2023 Fig.4 の近似）:

```
p_depol_threshold(p_e) ≈ 1.5% × (1 - p_e/24.9%)  [線形近似]
```

p_e = 20.4% のとき:
```
p_depol_threshold ≈ 1.5% × (1 - 20.4/24.9) ≈ 1.5% × 0.18 ≈ 0.27%
```

**警告**: 線形近似では動作点が閾値外に見える。ただし、実際の閾値境界は線形ではなく凸曲線であり、消失に対してより寛容である。正確な判定にはStimシミュレーションが必須。

**リスク評価**:
- SHYPS: 閾値境界付近で動作しており、マージンが狭い。Phase 1のStim検証が最重要マイルストーン。
- 4D HPC: スタビライザ重みが低い分、脱分極閾値がやや高く、マージンが改善する可能性あり。
- 表面符号: 脱分極閾値3.3%に対し0.81%であり、十分なマージン（4倍）がある。消失閾値もp_e_th ≈ 24.9%に対し20.4%で4.5%のマージン。

#### 5.6.4 Phase 1での検証計画

| 検証ID | 内容 | 成功基準 | 所要期間 |
|---|---|---|---|
| V1 | Stimノイズモデルの実機較正 | 実測ノイズパラメータとモデルの乖離 < 20% | 6ヶ月 |
| V2 | SHYPS d=4 の論理誤り率測定 | p_L(d=4) < 10⁻² | 3ヶ月 |
| V3 | SHYPS d=4→6 のスケーリング確認 | p_L(d=6)/p_L(d=4) < 0.1 | 3ヶ月 |
| V4 | デコーダレイテンシ実測 | < 1μs @ d=8 | 3ヶ月 |
| V5 | 4D HPC Stimシミュレーション | 実測ノイズモデルでの閾値確認 | 2ヶ月 |

**判定フロー**:
```
V1合格 → V2合格 → V3合格 → V4合格 → SHYPS採用（Phase 2へ）
                                        ↓ 不合格
                              V5合格 → 4D HPC採用
                                        ↓ 不合格
                              表面符号採用（論理qubit数を削減）
```
